# Listas, Sets, Maps, Pair i Triple

Kada tipu di kolesan ta rezolve un problema diferenti: **órden** (`List`), **únisidadi** (`Set`), ou **par-di-valor** (`Map`, `Pair`, `Triple`). Gosi nu ta odja kada un — kumo kria-l, kumo ler si konteúdu, i kual problema ki el ta rezolve midjor.

<SectionHeading variant="concept" seq={1}>List: Un Kolesan Ordenadu</SectionHeading>

**`List`** ta rapresenta un lista ordenadu di elementu. Se órden ta fikadu, i mesmu elementu pode aparese más di un bez. Bu ta uza `List` kuandu órden importa — sima un lista di kompras ou un playlist di múzika.

Imajina un estanti di livru, ku prateladu numeradu di 0 pa riba: kada livru ten se própriu lugar, i bu ta atxa-l dretu si bu sabe se númeru di prateladu (índisi). Bu pode adisiona más livru, tra un fora, ou muda un di lugar, ma órden ta fika — prateladu 0 sénpri ta bin antis di prateladu 1. `List` ta funsiona di mesmu manera: kada elementu ta guardadu na se própriu índisi, kumesa di zero.

Pa kria un `List`, uza funsan **`listOf`**:

```kotlin
fun main() {
    val kompras = listOf("arus", "fejon", "leti", "arus")

    println(kompras)          // [arus, fejon, leti, arus]
    println(kompras[0])       // arus
    println(kompras[2])       // leti
    println(kompras.size)     // 4
}
```

Tipu di rezultadu é **`List<T>`**, unde `T` é tipu di elementu — na izemplu di riba, `List<String>`. Bu ta asesa un elementu pa se índisi ku kolxetis `[]` (primeru índisi é `0`), i bu ta ler total di elementu ku propriedadi `size`.

### Verifika si un List Kontém un Elementu

Uza `contains` ou operador `in` pa verifika si un elementu sta dentu di lista:

```kotlin
fun main() {
    val kompras = listOf("arus", "fejon", "leti", "pon")

    println(kompras.contains("fejon"))    // true
    println(kompras.contains("asukra"))   // false
    println("pon" in kompras)             // true
    println("asukra" !in kompras)         // true
}
```

`!in` ta verifika kontráriu — si elementu **ka** sta dentu.

### Itera i Uza List Mutável

Pa pasa pa tudu elementu di un lista, uza un `for`-loop. I si bu meste adisiona ou tira elementu dipos di kriason, uza **`mutableListOf`** en bez di `listOf` — rezultadu é un `MutableList` ku metodu sima `add`, `remove` i asinason pa índisi:

```kotlin
fun main() {
    val kompras = listOf("arus", "fejon", "leti")
    for (item in kompras) {
        println("Meste kumpra: $item")
    }

    val listaMutável = mutableListOf("arus", "fejon")
    listaMutável.add("pon")
    println(listaMutável)              // [arus, fejon, pon]

    listaMutável[0] = "arus integral"
    println(listaMutável)              // [arus integral, fejon, pon]

    listaMutável.remove("fejon")
    println(listaMutável)              // [arus integral, pon]
}
```

:::callout{type=tip}
`val kompras = mutableListOf(...)` ka é kontraditóriu: `val` ta garanti ki `kompras` sénpri ta apunta pa **mesmu** `MutableList` — el ka pode resebe un lista diferenti dipos. Ma **konteúdu** di kel lista pode muda (`add`/`remove`/`[i] = ...`), pamodi kel é própriu ideia di `MutableList`.
:::

### Kria un List Novu Sen Muda Kel Orijinal: `+`

Un `List` normal (sen `mutable`) é **imutável** — ma isu ka signifika ki bu ka pode "adisiona" algu. Kombina `+` (un elementu, ou un otu `List`) ta kria un `List` **novu**, sen toka kel orijinal:

```kotlin
fun main() {
    val basi = listOf("Arus", "Fejon")
    val kompletu = basi + "Katxupa"
    println(basi)      // [Arus, Fejon] — orijinal ka mudou
    println(kompletu)  // [Arus, Fejon, Katxupa] — list novu

    val otu = listOf("Peixi") + listOf("Karni", "Fruta")
    println(otu) // [Peixi, Karni, Fruta]
}
```

:::callout{type=tip}
`+` (kria un list novu) i `mutableListOf(...).add(...)` (muda list ezistenti) ta rezolve mesmu problema di forma diferenti. Uza `+` kuandu bu meste guarda lista orijinal intakta; uza `mutableListOf` kuandu bu meste modifika lista ripitidamenti (sima un karrinhu di kompras ki ta kresi ku tenpu).
:::

### Array vs List: Kual Skodje?

Kotlin tanbe ten `Array<T>`, un otu tipu di kolesan ordenadu — bu ta kria-l ku `arrayOf`. Na primeru odjada, `Array` parsidu mutu ku `List`, ma ten un diferensa importanti: **`Array` ten tamanhu fixu** — bu ka pode adisiona ou tira elementu dipos di kriadu, só muda valor ku índisi (`[i] = ...`).

```kotlin
fun main() {
    val numerus = arrayOf(1, 2, 3)
    numerus[0] = 10          // OK — muda valor pa índisi
    println(numerus[0])      // 10
    println(numerus.size)    // 3 — tamanhu ka ta muda nunka
}
```

Bu ta repara `Array` na un lugar familiar: `fun main(args: Array<String>)` — parámetru ki ta resebe argumentus di linha di komandu, un padron erdadu di Java.

:::callout{type=tip}
Na prátika Kotlin, `List`/`MutableList` é kuazi sénpri skoja midjor — el é más flexível (pode kresi/diminui) i ten txeu metodu úteis. Guarda `Array` na memória pa kazu spesífiku sima `main(args: Array<String>)`, ou kuandu bu sabe ku serteza ki tamanhu ka ta muda nunka.
:::

<SectionHeading variant="concept" seq={2}>Set: Únisidadi Garantidu</SectionHeading>

Imajina númerus di un bilheti di lotaria: kada númeru sô ta aparse un bez — ka ten repetisan — i órden ki es sta skrebedu ka importa pa verifika si bu grava. Kel é izatamenti abstrasan matemátiku di **set** (konjuntu): un grupu di elementu úniku, sen repetisan, unde órden jeralmenti ka ten signifikadu.

**`Set`** ta iplementa kel mesmu ideia — é parsidu ku `List`, ma ku un diferensa fundamental: **se elementu ten ki ser úniku**. Si bu tenta adisiona un elementu ki dja sta la, Kotlin só ta ignora-l — sen erru.

Pa kria un `Set`, uza funsan **`setOf`**:

```kotlin
fun main() {
    val ilhas = setOf("Sal", "Santiago", "Fogo", "Sal")

    println(ilhas)          // [Sal, Santiago, Fogo]
    println(ilhas.size)     // 3 -- "Sal" repetidu ka ta konta dos bez
}
```

Tipu di rezultadu é **`Set<T>`**. Nota ki `Set` **ka ta garanti órden** — diferenti di `List` — i pamodi kel, bu **ka pode** asesa elementu pa índisi (`ilhas[0]` ka ta kompila pa un `Set`).

### Verifika si un Set Kontém un Elementu

Mesmu padron di `List`: uza `contains` ou `in`.

```kotlin
fun main() {
    val ilhas = setOf("Sal", "Santiago", "Fogo")

    println(ilhas.contains("Fogo"))    // true
    println(ilhas.contains("Brava"))   // false
    println("Sal" in ilhas)            // true
    println("Brava" !in ilhas)         // true
}
```

### Únisidadi i Igualdadi

Kumo ki Kotlin disidi si dos elementu é "mesmu valor", pa disidi si un é duplikadu? El ta uza igualdadi (`==`). Pa **klasis normal**, igualdadi é di **referénsia** — dos objetu só é igual si é **literalmenti mesmu objetu na memória**. Pa un **`data class`** (nu ta odja-l ku más detalhi na Módulu 5), igualdadi é **estrutural** — dos objetu ku mesmu valor é igual, mesmu ki es é dos objetu diferenti.

### Set Mutável

Sima `List`, `Set` tanbe ten se versan mutável — **`mutableSetOf`**:

```kotlin
fun main() {
    val ilhasVizitadu = mutableSetOf("Sal", "Santiago")
    ilhasVizitadu.add("Fogo")
    ilhasVizitadu.add("Sal")          // ka ta muda nada -- "Sal" ja sta la
    println(ilhasVizitadu)            // [Sal, Santiago, Fogo]

    ilhasVizitadu.remove("Santiago")
    println(ilhasVizitadu)            // [Sal, Fogo]
}
```

`+` tanbe funsiona pa `Set` — mesmu padran di `List` (kria un `Set` novu, sen muda kel orijinal), ma **ta respeita únisidadi**: si elementu adisionadu dja sta la, `Set` novu ka ta duplika-l.

```kotlin
fun main() {
    val ilhas = setOf("Sal", "Boavista")
    val maisIlhas = ilhas + "Sal" // "Sal" dja sta la — ignoradu
    println(maisIlhas) // [Sal, Boavista]
}
```

<SectionHeading variant="concept" seq={3}>Map: Par Txavi-Valor</SectionHeading>

Imajina un livru di kontatus: bu ta prokura nomi di un pesoa i atxa se númeru di telefoni dretu, sen bu meste pasa página pa página até atxa-l. **`Map`** ta funsiona di mesmu manera: é un kolesan di **par txavi-valor** (*key-value*) — bu da un **txavi** (`key`), kumo un nomi, i el ta debolve **valor** (`value`) korespondenti, kumo un númeru di telefoni. Txavi ten ki ser úniku, ma valor pode ripiti. Bu ta uza `Map` kuandu bu meste liga dos informasan — sima nomi di disiplina ku nota.

Pa kria un `Map`, uza funsan **`mapOf`** ku infix **`to`** pa kada par:

```kotlin
fun main() {
    val notas = mapOf("Matemátika" to 18, "Kriolu" to 16, "Inglés" to 15)

    println(notas)                  // {Matemátika=18, Kriolu=16, Inglés=15}
    println(notas["Kriolu"])        // 16
    println(notas["Fransés"])       // null -- "Fransés" ka sta na map
}
```

Tipu di rezultadu é **`Map<K, V>`**. Nota ki buska un valor pa un txavi ku `[]` ta debolve un tipu **nulável** (`Int?` na es izemplu) — si txavi ka izisti, rezultadu é `null`.

Ma kuzé ki ta kontise si bu skrebe **mesmu txavi dos bez** kuandu bu ta kria un `Map`?

```kotlin
val notas = mapOf("Kriolu" to 16, "Kriolu" to 18)
println(notas)        // {Kriolu=18}
println(notas.size)   // 1 -- txavi ripitidu: últimu valor ta sobrevive
```

Si bu skrebe mesmu txavi dos bez, últimu valor ta sobrevive (override) — kel é mesmu padron di únisidadi ki `Set` ta uza, ma pa txavi.

:::callout{type=warning}
Diferenti di `List`/`Set` (`contains`), buska un valor na `Map` ku `notas["Fransés"]` **nunka** ta da un erru — el só ta debolve `null` si txavi ka izisti. Sénpri konsidera kel `null` na bu kódiku — sima ku un tipu nulável kualker.
:::

### Itera un Map i Verifika Txavis/Valor

Pa itera un `Map`, bu ta pasa pa se pares na un `for`-loop, i Kotlin ta permiti **destruturason** diretu na dos variável (`txavi`, `valor`). Bu tanbe pode verifika si un txavi ou un valor spesífiku sta dentu di `Map`, ku `containsKey` i `containsValue`:

```kotlin
fun main() {
    val notas = mapOf("Matemátika" to 18, "Kriolu" to 16, "Inglés" to 15)

    for ((materia, nota) in notas) {
        println("$materia: $nota")
    }

    println(notas.containsKey("Kriolu"))     // true
    println(notas.containsKey("Fransés"))    // false
    println(notas.containsValue(15))         // true
}
```

### Map Mutável

Sima `List`/`Set`, `Map` tanbe ten se versan mutável — **`mutableMapOf`**. Bu pode adisiona un par ku `[]` ou ku `put`, i tira un par ku `remove`:

```kotlin
fun main() {
    val notas = mutableMapOf("Matemátika" to 18, "Kriolu" to 16)

    notas["Inglés"] = 15          // adisiona un novu par
    notas.put("Istória", 17)      // otu manera di adisiona

    println(notas)                // {Matemátika=18, Kriolu=16, Inglés=15, Istória=17}

    notas.remove("Kriolu")
    println(notas)                // {Matemátika=18, Inglés=15, Istória=17}
}
```

Pa un `Map` imutável, `+` ta adisiona (ou sobreskrebe) un par i kria un `Map` **novu**; `-` ta tira un txavi, i tanbe kria un `Map` novu:

```kotlin
fun main() {
    val notas = mapOf("Kriolu" to 16, "Matemátika" to 14)
    val ku18 = notas + ("Inglés" to 18)
    println(notas)  // {Kriolu=16, Matemátika=14} — orijinal ka mudou
    println(ku18)   // {Kriolu=16, Matemátika=14, Inglés=18}

    val sénKriolu = notas - "Kriolu"
    println(sénKriolu) // {Matemátika=14}
}
```

<SectionHeading variant="concept" seq={4}>Pair i Triple: Kombina Valor Sen Kria un Klasi Novu</SectionHeading>

Kuandu bu só meste guarda **dos** ou **tres** valor djuntu — sen presiza defini un `data class` kompletu — Kotlin ta da **`Pair`** i **`Triple`**, prontu pa uza.

**`Pair`** ta guarda dos valor, asesível pa `first` i `second`. Bu pode kria-l ku konstrutor `Pair(...)` ou ku infix `to` (mesmu `to` ki bu dja uzadu pa `Map`). **`Triple`** ta guarda tres valor — `first`, `second` i `third`.

Ambos pode ser **destruturadu** diretu na variável separadu:

```kotlin
fun main() {
    val koordenada = Pair(14.9, "Praia")
    println(koordenada.first)     // 14.9
    println(koordenada.second)    // Praia

    val (latitudi, sidadi) = koordenada
    println(latitudi)             // 14.9
    println(sidadi)               // Praia

    val koordenadaTo = 14.9 to "Praia"   // 'to' tanbe ta kria un Pair
    println(koordenadaTo == koordenada)  // true

    val produtu = Triple("Arus", 5, 450.0F)
    println(produtu.first)        // Arus
    println(produtu.second)       // 5
    println(produtu.third)        // 450.0

    val (nomi, kantidadi, presu) = produtu
    // presu é di tipu Float -- ka Double, pamodi el foi kriadu ku sufixu F
    println("$nomi: $kantidadi unidadi, $presu ECV")
}
```

:::callout{type=tip}
`Pair` i `Triple` é bon pa un rezultadu rápidu i lokal — sima un funsan ki debolve dos valor djuntu. Kuandu bu meste más di tres valor, ou kuandu bu meste nomi klaru pa kada propriedadi, un `data class` (Módulu 5) é midjor skoja.
:::
