# Funsons: Argumentus i Scope

Imajina un funsan `kria(true, false, 10, "kea")`. Kual parámetru é kual? Sen odja definisan di funsan, é imposível sabe. Kuandu un funsan ten txeu parámetrus, Kotlin ta da-bu dos ferramentas pa rezolve kel: **argumentus padron** i **argumentus nomeadu**. Dipos, nu ta odja kumo Kotlin ta disidi unde kada variável i funsan pode ser uzadu — se **scope**.

<SectionHeading variant="concept" seq={1}>Definindo un Funsan</SectionHeading>

Bu ja ta uza funsons di lisons anterior — `println`, i mesmu bu própi funsons sima `verificaIdadi` ou `fatorial`. Antis di odja argumentus padron i nomeadu, nu ta relembra sintaksi bázika: kumo defini un funsan di kabu na rabu.

Pensa na un funsan sima un **kaxa preta**: bu po kuzê dentu (se parámetrus), el ta faze algu ku es — sen bu mesti sabe tudu detalhi di kuzê ki ta kontise la dentu — i el ta dá-u un rezultadu. Ou pensa nel sima un **reseta**: ingredientis ta entra, pasus di reseta ta korri (kel é kórpu di funsan), i un pratu ta sai (kel é `return`). Vantaji di un reseta é ki bu pode uza-l kuantas bez bu kre, ku ingredientis diferenti kada bez — sen skrebe-l di novu. É izatamenti kel ki nu ta odja gosi: mesmu funsan `soma`, ma ku parámetrus diferenti kada txamada.

Nin tudu funsan meste devolve un valor. Kuandu un funsan só ta **faze algu** (sima imprimi na terminal) sen devolve nada, bu ka skrebe tipu di retornu — Kotlin ta atribui `Unit` automatikamenti, un tipu spesial ki sinifika "sen valor útil":

```kotlin
fun sauda(nomi: String) {
    println("Ola, $nomi!")
}
```

`sauda` ka ten `: Tipu` dipos di se paréntesis — se tipu di retornu é `Unit` (implísitu), i el ka ten `return` ku valor nenhun.

<SectionHeading variant="concept" seq={2}>Argumentus Padron</SectionHeading>

Un funsan pode difini un valor padron pa un parámetru. Si kenha ta txoma funsan ka indika kel argumentu, Kotlin ta uza valor padron sózin — kumo un reseta ki ja ten kantidadi disidida pa un ingredienti, ma sénpri pronta pa bu troka-l si bu kre otu kuza.

<GlossaryText
  text="Es sintaksi — `parámetru: Tipu = valorPadron` — ta txoma-s un [[argumentu padron]]."
  terms={{
    "argumentu padron": { en: "default argument", definition: "Un valor ki un parámetru ta risebe kuandu txamada ka indika kel argumentu spesifikamenti." },
  }}
/>

```kotlin
fun kumprimenta(nomi: String, formal: Boolean = false) {
    println(if (formal) "Bon dia, $nomi." else "Ola, $nomi!")
}

fun main() {
    kumprimenta("Telma")        // Ola, Telma!
    kumprimenta("Zeca", true)       // Bon dia, Zeca.
}
```

`formal` ten valor padron `false`. Kuandu bu ka indika-l — sima na `kumprimenta("Telma")` — Kotlin ta atribui `false` sózin, i bu ka mesti skrebe dos versons diferenti di mesmu funsan.

Bu pode difini valor padron pa kantu parámetru bu kre, i kombina-s ku parámetrus sen valor padron:

```kotlin
fun apresenta(nomi: String, papel: String = "Estudanti", turma: String = "Kotlin") {
    println("$nomi é $papel na turma $turma.")
}

fun main() {
    apresenta("Telma")                  // Telma é Estudanti na turma Kotlin.
    apresenta("Zeca", "Monitor")     // Zeca é Monitor na turma Kotlin.
}
```

<SectionHeading variant="concept" seq={3}>Argumentus Nomeadu</SectionHeading>

Dipos di un tempu, `apresenta("Telma", "Monitor")` ka é klaru sen le definisan di funsan un otu bez — kual argumentu é `nomi` i kual é `papel`? Ku **argumentus nomeadu**, bu skrebe nomi di parámetru na txamada mesmu, i txamada ta splika-se sózin:

<GlossaryText
  text="Es forma di txamada ta txoma-s un [[argumentu nomeadu]]."
  terms={{
    "argumentu nomeadu": { en: "named argument", definition: "Un argumentu pasadu ku sintaksi `parámetru = valor` na txamada, ki ta liga se valor ku nomi di parámetru — ka ku se pozisan." },
  }}
/>

```kotlin
fun main() {
    apresenta(nomi = "Telma", papel = "Monitor")
    apresenta(papel = "Estudanti", nomi = "Zeca")   // órden ka importa kuandu bu nomeia
}
```

A vantaji más grandi di argumentus nomeadu é ki bu pode **salta parámetrus anterior**: bu só indika kes ki bu kre muda, i restu ta risebe se valor padron.

```kotlin
fun main() {
    apresenta(nomi = "Telma", turma = "Kotlin Avansadu")
    // Telma é Estudanti na turma Kotlin Avansadu.
    // (papel ka foi indikadu — ta risebe se valor padron "Estudanti")
}
```

Un só funsan, tres forma di txama-l ku mesmu resultadu:

**Posizional (tudu argumentus)**

```kotlin
fun saudaKu(saudason: String = "Ola", nomi: String = "Mundu") {
    println("$saudason, $nomi!")
}

fun main() {
    saudaKu("Bon dia", "Telma")
}
// Bon dia, Telma!
```

**Nomeadu (salta primeru pozisan)**

```kotlin
fun main() {
    saudaKu(nomi = "Zeca")
}
// Ola, Zeca!
// (saudason ka foi indikadu — ta risebe se valor padron "Ola")
```

**Nomeadu (órden trokadu)**

```kotlin
fun main() {
    saudaKu(nomi = "Telma", saudason = "Odjadu")
}
// Odjadu, Telma!
```

:::callout{type=warning title="Regra: posizional antis di nomeadu"}
Bu pode mistura argumentus posizional ku argumentus nomeadu na mesmu txamada, ma tudu argumentu posizional ten ki ben **antis** di kualker argumentu nomeadu. Un bez ki bu kumesa uza un argumentu nomeadu, tudu argumentu ki ben dipos di el debe ser nomeadu tanbe — `apresenta(papel = "Monitor", "Telma")` **ka ta kompila**, pamodi `"Telma"` (posizional) ta ben dipos di `papel` (nomeadu).
:::

<SectionHeading variant="concept" seq={4}>Argumentus Variável: vararg</SectionHeading>

Até gosi, kada funsan ten un número fixu di parámetru. Ma i si bu ka sabe kantu argumentu ta ser pasadu — sima `println` ki pode risebe un ou vários valor? Kotlin ta da `vararg` pa kel: un parámetru ki pode risebe **zeru ou más** valor, tudu ku mesmu tipu.

  notes={[
    { m: 1, title: "vararg", body: "`vararg numerus: Int` ta permiti txoma `soma` ku 0, 1 ou kantu `Int` bu kre — tudu na un só parámetru." },
    { m: 2, title: "Dentu di funsan, é un Array", body: "Dentu di `soma`, `numerus` ta komporta sima un `Array<Int>` — bu pode itera-l ku `for`." },

Un funsan só pode ten **un** parámetru `vararg`, i el normalmenti ta ben pa último (a menus ki bu uza argumentus nomeadu pa kes ki ben dipos).

:::callout{type=tip}
Bu ja ta uza un funsan ku `vararg` desdi Lisan 1: `println` é un izemplu — el pode risebe zeru ou vários argumentu.
:::

<SectionHeading variant="concept" seq={5}>Scope di Variável: Lokal vs Top-Level</SectionHeading>

Kada bez ki bu difini un variável dentu di un funsan, kel variável só é vizível dentu di se scope: as txavis `{ }` unde el foi definidu. Kel scope ta txoma-s **scope lokal**, i variável, **variável lokal**.

```kotlin
fun kalkulaImpostu(): Double {
    val taxa = 0.15          // variável lokal — só vizível dentu di kalkulaImpostu()
    return 1000 * taxa
}

fun kalkulaDeskontu(): Double {
    val taxa = 0.05          // otu variável lokal, mesmu nomi — sen konflitu ku `taxa` di `kalkulaImpostu()`
    return 1000 * taxa
}
```

Es dos variáveis txoma `taxa`, ma ka ten konflitu — kada un só izisti dentu di se própriu funsan. Si bu tenta uza `taxa` fora di `kalkulaImpostu()`, Kotlin ta da un eru di kompilasan: `Unresolved reference`.

Si bu kre un variável ki tudu funsan pode partilha, bu debe difini-l **fora** di tudu funsan — na nível más altu di fixeru. Kel ta txoma-s un **variável di top-level**, i el é vizível pa tudu kel fixeru.

```kotlin
var saldu = 0                        // variável di top-level — vizível pa tudu funsan

fun adisionaSaldu(montanti: Int) {
    saldu += montanti
}

fun mostraSaldu() {
    println("Saldu: $saldu")
}

fun main() {
    adisionaSaldu(500)
    adisionaSaldu(250)
    mostraSaldu()
}
```

<SectionHeading variant="concept" seq={6}>Funsons Lokal</SectionHeading>

Sima variáveis, funsons tanbe pode ser lokal. Un **funsan lokal** é un funsan definidu **dentu** di otu funsan. El só é vizível dentu di kel funsan, i só dipos di se pontu di definisan.

```kotlin
fun main() {
    fun sauda(nomi: String) {          // funsan lokal — só vizível dentu di main()
        println("Ola, $nomi!")
    }

    sauda("Telma")                 // OK — sauda() ja sta definidu li riba
    sauda("Zeca")
}

fun otuFunsan() {
    // sauda("Telma")   // ERU: Unresolved reference — sauda() ka é vizível li fora
}
```

Repara na diferensa ku funsons di top-level: es últimu pode ser txomadu di kualker lugar na fixeru, mesmu antis di se definisan — Kotlin ta odja tudu funsan di top-level di un bez. Ma un funsan lokal sigi mesmu regra ki un variável lokal: el só izisti dipos di linha unde foi definidu.

:::callout{type=warning title="Órden importa pa funsons lokal"}
Diferenti di funsons di top-level, un funsan lokal debe ser definidu **antis** di se primeru txamada — mesmu dentu di mesmu funsan. Tenta txoma-l antis di se definisan ta da un eru: `Unresolved reference`.
:::

Kada bez ki bu skrebe un funsan ku txeu parámetru, ou meste kombina vários pedasu di kódiku, es ferramentas — argumentus padron, argumentus nomeadu, `vararg`, i funsons lokal — ta faze bu kódiku más klaru i más fásil di manutené.

Gosi é bu bez: nu ta pratika argumentus padron ku argumentus nomeadu, i rezolve un eru di scope.
