# Booleans i Nulidadi Sigura

Bu ta sai brinka só si tenpu sta bon **i** bu ten tenpu livri — dos kondisan meste ser verdadi na mesmu tenpu. Ma si tenpu ka sta bon **ou** bu ka ten tenpu livri, bu ta fika kaza. Na vida real, nos ta toma desizan baseadu na kondisons a tudu momentu — i programas ta funsiona di mesmu manera. Na Kotlin, kada kondisan ta pasa pa un tipu simpli: **Boolean**, ku só dos valor posível — `true` ou `false`.

:::callout{type=warning}
Si bu ben di Python: la, `True` i `False` ta skrebe ku primeru letra maiúskula. **Na Kotlin é kontráriu** — só `true` i `false` (tudu minúskulu) ta izisti. `True`/`False` (maiúskulu) na Kotlin ta da `Unresolved reference`, ka un valor Boolean.
:::

<SectionHeading variant="concept" seq={1}>Operadoris di Komparasan</SectionHeading>

Operadoris di komparasan ta kompara dos valor i ta devolve senpri un `Boolean`: `true` ou `false`.

<CodeExample position={0} />

`==` ta pergunta "es dos valor é igual?"; `!=` ta pergunta kontráriu. `>`, `<`, `>=` i `<=` ta kompara órden entri valor numériku.

### `==` vs `===`: Igualdadi ku Identidadi

`==` (i se kontráriu `!=`) ta kompara **valor** — kel ki bu ja odja li riba. Kotlin tanbe ten un par di operador relasionadu, `===` i `!==`, ki ta kompara **identidadi**: si dos variavel ta apunta pa **mesmu objetu na memória**, ka só si es ten mesmu valor.

```kotlin
val a = 5
val b = a
println(a == b)   // true — mesmu valor
println(a === b)  // true — b é literalmenti mesmu a
```

Pa tipus simpli, `==` i `===` normalmenti ta da mesmu rezultadu. Diferensa ta fika klara só kuandu nu ta modela nos própriu klasses (Módulu 5) — la, dos objetu **diferenti** ku mesmu dadus pode ser `==` ma `===` `false`.

<SectionHeading variant="concept" seq={2}>Operadoris Lójiku: &&, || i !</SectionHeading>

Operadoris lójiku ta kombina vários kondisons Boolean nun só rezultadu.

### `&&` — AMBOS lado meste ser `true`

<CodeExample position={1} />

`&&` só ta devolve `true` kuandu **dos** lado é `true`. Na izemplu di riba, `podeEntra` só é `true` pamodi `idadi >= 18` **i** `tenBilheti` é `true` — falta un só, i rezultadu ta `false`.

### `||` — só UN lado meste ser `true`

<CodeExample position={2} />

`||` é más flexível: el só ta devolve `false` kuandu **dos** lado é `false`. Na izemplu, `tenDiskontu` é `true` mesmu si `studanti` é `false`, pamodi `idosu` é `true`.

### `!` — inverte `true`/`false`

```kotlin
fun main() {
    println(!true)   // false
    println(!false)  // true
}
```

`!` ta pega un Boolean i devolve kontráriu.

<SectionHeading variant="concept" seq={3}>Presedensia: `!` Antis di `&&` i `||`</SectionHeading>

Kuandu bu kombina `!` ku `&&`/`||` sen paréntesis, é fásil interpreta trotu — `!` só ta nega valor **imediatamenti** dipos di el, ka rezultadu di ekspresan interu.

    { m: 2, title: "Dipos `&&` kombina", body: "Ku `menorDiIdadi` já negadu, `&&` ta kombina rezultadu ku `tenBilheti`." },
    { m: 3, title: "Rezultadu", body: "Kumo dos lado é `true`, `podeEntra` é `true`." },

/>

:::callout{type=warning}
`!` ten presedensia más altu ki `&&` i `||` — el só ta nega valor imediatamenti dipos di el. Na dúvida sobri órden, uza **paréntesis** pa fika klaru: `!(menorDiIdadi && tenBilheti)` é diferenti di `!menorDiIdadi && tenBilheti`.
:::

<SectionHeading variant="concept" seq={4}>Nulidadi Sigura (Null Safety)</SectionHeading>

Na Kotlin, un variavel di tipu `String` (sen `?`) **nunka** pode guarda `null` — el é **non-null pa padron**.

```kotlin
var mensajen: String = "Ola Mundu!"
mensajen = null // eru di kompilasan: 'null' ka pode ser valor di un tipu non-null
```

Kel garantia ta iziste pamodi un `NullPointerException` (NPE) é un di errus más kumun na txeu linguajen — el ta kontise kuandu bu tenta asesa un propriedadi ou metodu di un valor ki é `null`. Kotlin ta previni NPE **na kompilasan**, antis ki bu programa mesmu kore.

Un tipu nulável (`String?`) **ka é mesmu ki** un valor padron ou opsional. El só signifika ki bu variavel **pode** guarda `null` — bu ainda meste verifika-l (ku `?.` ou `if`) antis di uza.
:::

<SectionHeading variant="concept" seq={5}>Operador Elvis: `?:` — un Valor Padron Kuandu É `null`</SectionHeading>

`?.` ta proteje kontra NPE, ma se rezultadu inda pode ser `null` — sima bu ja odja ku `mensajen?.length` li riba. Txeu bez, bu ka kre `null` sima rezultadu final: bu kre un **valor padron** na se lugar. Pa kel, Kotlin ta da **operador Elvis** (`?:`).

<GlossaryText
  text="Ekspreson `a ?: b` ta ler-se: si `a` ka é `null`, uza `a`; sinon, uza `b`. Es é [[operador Elvis]]."
  terms={{
    "operador Elvis": { en: "Elvis operator", definition: "Ekspreson `?:` ki ta devolve valor a skerda si el ka é `null`, ou valor a direita (padron) si el é `null`." },
  }}
/>

<AnnotatedCode
  lang="kotlin"
  title="`?:` ta trabadja djuntu ku `?.`"
  code={[
    { t: 'var mensajen: String? = null', m: 0 },
    { t: 'val tamanhu = mensajen?.length ?: 0', m: 1 },
    { t: 'println(tamanhu) // 0', m: 2 },
  ]}
  notes={[
    { m: 1, title: "Kombinasan `?.` + `?:`", body: "`mensajen?.length` só é `null` pamodi `mensajen` é `null`. `?:` ta pega kel `null` i ta troka-l pa `0` — valor padron a direita." },
    { m: 2, title: "Nunka `null`", body: "`tamanhu` gosi é senpri un `Int` — nunka `Int?`. `?:` ta poupa-bu di verifika `null` kada bez ki bu ta uza `tamanhu`." },
  ]}
/>

Kompara kes dos kaminhu pa trata `null`:

```kotlin
// Só ku ?. — rezultadu pode ainda ser null
var apelidu: String? = null
println(apelidu?.uppercase()) // null

// Ku ?. + ?: — rezultadu NUNKA é null
println(apelidu?.uppercase() ?: "SEN APELIDU") // SEN APELIDU
```

:::callout{type=tip}
`?:` ka só funsiona ku `?.` — el ta uzável ku **kualker** ekspresan nulável, ka só txamadas. Bu ta repara `?:` otu bez más pa frenti nes kursu, senpri ku es mesmu funsan: "kel valor si el ka é `null`, sinon, es otu."
:::

### `?.let { }`: Faze Algu Só Si Ka É `null`

Bu ja sabe `?.` pa asesa un propriedadi/metodu di manera sigura, i `?:` pa da un valor padron. Ma i si bu meste **faze vários koza** ku un valor só kuandu el ka é `null`? Pa kel, Kotlin ta da `let`, kombinadu ku `?.`.

<AnnotatedCode
  lang="kotlin"
  title="`?.let { }` — só ta izekuta si valor ka é null"
  code={[
    { t: 'var apelidu: String? = "Djiku"', m: 1 },
    { t: "apelidu?.let {", m: 2 },
    { t: '    println("Apelidu: $it")', m: 3 },
    { t: '    println("Tamanhu: ${it.length}")', m: 3 },
    { t: "}", m: 0 },
  ]}
  notes={[
    { m: 1, title: "Variavel nulável", body: "`apelidu` pode guarda `null`, ma gosi el ten un valor." },
    { m: 2, title: "`?.let { ... }`", body: "`let` ta resebe bloku `{ ... }` ki só ta izekuta si `apelidu` ka é `null`." },
    { m: 3, title: "`it`", body: "Dentu di `let`, `it` ta referensia valor — já **ka nulável** akí dentu, pamodi Kotlin sabe ki el ka é `null`." },
  ]}
/>

Si `apelidu` fosi `null`, bloku dentu di `let` ka ta izekuta — el ka ta imprimi nada, sen erru.

:::callout{type=tip}
`?.let { ... }` é útil kuandu bu meste **más di un linha di kódiku** ku un valor nulável. Pa un só ekspresan ku valor padron, `?:` sózin (sima nu odja li riba) kontinua ser midjor skoja.
:::

<SectionHeading variant="concept" seq={6}>Armadidjas Komun</SectionHeading>

```kotlin
// ARMADIDJA 1: '!' ta liga más fórti ki '&&' i '||'
println(!true && false)   // false — só 'true' ta ser negadu, dipos && ku false
println(!(true && false)) // true  — paréntesis muda tudu

// ARMADIDJA 2: tipu nulável ka é un "valor padron"
var kodigu: String? = null   // 'null' é un valor real — ka é 'omitidu' ou 'vaziu'
println(kodigu?.length)      // null — TEN ki verifika antis di uza
```

Gosi ki bu sabe kombina kondisons ku operadoris di komparasan i lójiku, i trata `null` di manera sigura, bu sta pronto pa kontrola fluxu di bu programa ku `if` i `when`.
