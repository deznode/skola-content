# Kuzé ki é Kotlin?

Bu kaba di prende bu primeru linguajen di programasan. Gosi, bu sta li pa kumesa un novu — **Kotlin**. Un pergunta natural bu pode ta faze: pamodi meste prende otu linguajen, si dja bu sabe programa?

Resposta é sinplis: kada linguajen ta abri portas diferenti. Kotlin ta introduzibu a un di kes ekosistema más grandi di dizenvolvimentu di software na mundu — di apps Android te sistema backend di grandi enpresa. I midjór di tudu: kes konseitu ki dja bu prende (variavel, tipu, kondisan, loop) ta transferi direitamenti pa Kotlin.

Kotlin é un linguajen di programasan modernu, kriadu pa **JetBrains** — un enpresa konxidu pa si ferramentas di dizenvolvimentu, sima IntelliJ IDEA.

<SectionHeading variant="concept" seq={2}>Kotlin i Java: Dos Linguajen, Un Só Mákina</SectionHeading>

Es é un di kes fatu más distintu di Kotlin: el ka é só "primu" di Java — el ta kore na **mesmu JVM**, i pode uza kódiku di Java direitamenti, sen konversan, sen adaptador.

Ma Python i JavaScript ka pode faze kel-li — kada un ta kore riba di un mákina virtual diferenti, ku formatu internu diferenti. Na prátika, kel signifika:

- Bu pode txoma un funsan Java dentu di un programa Kotlin.
- Bu pode txoma un funsan Kotlin dentu di un programa Java.
- Tudu biblioteka Java — i ten txeu, dezdi web te bazi di dadus — sta disponível pa Kotlin tanbe, sen trabadju extra.

Keli ta motiva txeu empresa ki dja ten sistema Java grandi sta ta adisiona Kotlin — es pode kombina dos linguajen dentu di mesmu prujetu, sen perde tudu kódiku di Java ki es dja ten.

<SectionHeading variant="concept" seq={4}>Pamodi Kotlin É Popular</SectionHeading>

Dos propriedadi ta distingui Kotlin di txeu otu linguajen:

- **Menus kódiku**: Kotlin ta skrebe menus kódiku pa faze mesmu kuza ki Java. Un programa ki na Java ta leba txeu linha, na Kotlin txeu bez ta leba menus di metadi.
- **Null safety**: Kotlin ta djuda evita un kategoria interu di eru — famozu `NullPointerException` (ou "NPE") — pamodi el ta obriga-bu deklara di manera klaru kual variavel pode ser `null` i kual ka pode.

Es dos propriedadi ka é só un splika kurtu — es ta paresi na tudu lisan di es kursu. Kada bez ki bu ta prende un konseitu novu (variavel, fluxu di kontrolu, funsan, koleson, ou klasis), bu ta odja kumo Kotlin ta faze-l di manera más kurtu i más siguru ki manera tradisional.

<SectionHeading variant="concept" seq={5}>Bu Purmeru Programa Kotlin</SectionHeading>

Tudu linguajen di programasan ten un tradisan: primeru programa ki bu ta skrebe é un ki ta imprimi un kumprimentu sinplis.

```kotlin
fun main() {
    println("Ola Mundu!")
}
```

`fun main()` é pontu di entrada di tudu programa Kotlin — é li ki tudu ta kumesa kuandu bu kore-l. `println(...)` ta imprimi un linha di testu na terminal.

Na Java, mesmu programa ta pidi txeu más:

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Ola Mundu!");
    }
}
```

Kotlin ta da mesmu rezultadu ku menus kódiku — un primeru gostu di menus kódiku.

Kotlin tanbe ta permiti inseri valor di variavel direitamenti dentu di un `String`, ku un template:

```kotlin
fun main() {
    val nomi = "Telma"

    println("Ola, $nomi! Bu sta na kaminhu di prende Kotlin.")
}
```
