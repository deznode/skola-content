# Klasses, Propriedadis i Eransa

Imajina bu ta konstrui un sistema pa un negósiu di **Aluguer di Veikulu** na Santiago. Kada karu ki bu ten pa aluga tene se própriu matrikula i se própriu presu pa dia — i bu meste guarda es informason di manera organizadu, pa dipos kria reservas pa klientis. Un `class` (klasse) ta da-bu kel: un moldi pa modela un entidadi real na kódiku, ku propriedadis ki ta guarda se dadus.

Na es lisan, nu ta kumesa ku un klasse `Veikulu`, dipos nu ta odja kumo `this` ta distingui un konfliktu di nomis, kumo un `data class` `Reserva` ta trata igualdadi i kópia di manera diferenti di un klasse normal, i pa fin kumo un klasse `Karu` pode erda di `Veikulu` ku eransa.

<SectionHeading variant="concept" seq={1}>Klasse, konstrutor primariu i propriedadis</SectionHeading>

Pa modela un veikulu di aluguer, nu ta kria un klasse `Veikulu` ku dos propriedadis: `matrikula` (String) i `presuPerDia` (Double). Kes dos propriedadis ta bai dirétamenti dentu di **konstrutor primariu** — lista di parámetrus ki ben logu dipos di nomi di klasse:

<SectionHeading variant="concept" seq={2}>Konstrutor Sekundáriu i Bloku init</SectionHeading>

Ás vezis bu meste izekuta kódiku **durante** kriason di un objetu — sima verifika si un valor é válidu — ou da **más di un manera** di kria un objetu. Kotlin ta da dos ferramenta pa kel: **bloku `init`** i **konstrutor sekundáriu**.

Un bloku `init` ta izekuta **na hora** ki un objetu é kriadu, dipos ki propriedadis di konstrutor primariu sta atribuídu:

<AnnotatedCode

    { t: "        println(\"Kriadu: $matrikula\")", m: 2 },
    { t: "    }", m: 0 },
    { t: "}", m: 0 },

  notes={[
    { m: 1, title: "init", body: "Kódiku dentu di `init { }` ta izekuta un só bez, na hora ki objetu é kriadu — dipos di `matrikula`/`presuPerDia` sta atribuídu." },
    { m: 2, title: "Uzu tipiku", body: "Validason (`require`) i log/setup inisial é uzu kumun di `init` — un klasse pode ten más di un bloku `init`, izekutadu na órden ki sta skrebedu." },

Un **konstrutor sekundáriu** ta da un otu manera di kria un objetu, ku parámetrus diferenti — ma el debe txoma konstrutor primariu (ou otu konstrutor sekundáriu) ku `this(...)`:

<AnnotatedCode
  lang="kotlin"
  title="Konstrutor sekundáriu: un presu padron"
  code={[
    { t: "class Veikulu(", m: 0 },
    { t: "    val matrikula: String,", m: 0 },
    { t: "    var presuPerDia: Double,", m: 0 },
    { t: ") {", m: 0 },
    { t: "    constructor(matrikula: String) : this(matrikula, 4000.0)", m: 1 },
    { t: "}", m: 0 },
    { t: "", m: 0 },
    { t: "fun main() {", m: 0 },
    { t: "    val karu1 = Veikulu(\"ST-45-AB\", 5000.0)  // konstrutor primariu", m: 2 },
    { t: "    val karu2 = Veikulu(\"ST-99-ZZ\")           // konstrutor sekundáriu", m: 3 },
    { t: "    println(karu2.presuPerDia)                // 4000.0", m: 3 },
    { t: "}", m: 0 },
  ]}
  notes={[
    { m: 1, title: "constructor(...) : this(...)", body: "Konstrutor sekundáriu ta resebe só `matrikula`, i txoma konstrutor primariu ku `this(matrikula, 4000.0)` — 4000.0 é presu padron." },
    { m: 2, title: "Konstrutor primariu", body: "Kuandu bu da dos argumentu, é konstrutor primariu ki ta izekuta." },
    { m: 3, title: "Konstrutor sekundáriu", body: "Kuandu bu da un só argumentu, é konstrutor sekundáriu ki ta izekuta, i el ta pasa 4000.0 sima presu padron." },
  ]}
/>

:::callout{type=tip}
Na txeu kazu, un konstrutor sekundáriu pode ser trokadu pa un **argumentu padron** na konstrutor primariu (sima `var presuPerDia: Double = 4000.0`, ki nu odja na Lisan 7). Uza konstrutor sekundáriu kuandu lójika di kriason é más kompleksu ki un só valor padron.
:::

<SectionHeading variant="concept" seq={3}>Receiver i `this`</SectionHeading>

Kuandu bu skrebe un métodu dentu di un klasse, es métodu ta ben ku un objetu implísitu — **receiver** — ki é instánsia ki ta txoma métodu. Bu pode referensia kel objetu explisitamenti ku `this`. Normalmenti bu ka meste skrebe `this.` pamodi Kotlin já ta asumi-l, ma ten un kazu unde el é **obrigatóriu**: kuandu un parámetru di métodu tene mesmu nomi ki un propriedadi di klasse.

<CodeExample position={0} />

Dentu di `atualizaPrecu`, parámetru `presuPerDia` ta "skonde" propriedadi ku mesmu nomi. Si bu skrebe só `presuPerDia`, Kotlin ta intende ku é parámetru — ka propriedadi di objetu. Pa referensia **propriedadi di objetu**, bu meste ser explisitu: `this.presuPerDia`.

:::callout{type=tip}
Bu ka meste skrebe `this.` sénpri — Kotlin ta asumi-l automatikamenti kuandu ka ten konfuzon di nomi. Uza-l só kuandu, sima li riba, un parámetru ta skonde un propriedadi.
:::

<SectionHeading variant="concept" seq={4}>Data classes: `Reserva`</SectionHeading>

Gosi nu meste modela un **reserva** (booking) — kual klienti ta aluga kual veikulu, i pa kantu dia. Un klasse normal ka ta kompara valoris di manera útil: dos instánsias ku mesmu dadus ka ta ser `==` un pa otu, i `println()` ta imprimi só un endereso di memória. Pa isu, Kotlin tene un modifikador `data`:

<CodeExample position={1} />

Nota dos kuza importanti li. `reserva1 == reserva2` é `true` pamodi `Reserva` é `data class` — igualdadi ta kompara **valor** di kada propriedadi, ka referensia. Ma `reserva1 == reserva3` é `false`! Mesmu ku `karu1` i `karu2` tene mesmu `matrikula` i mesmu `presuPerDia`, es é **duas instánsias diferenti** di `Veikulu` — i `Veikulu` ka é `data class`, nton el ka tene igualdadi estrutural própriu. Un `data class` só ta kompara si própriu propriedadis; el ka ta muda kumo otus klasses (sima `Veikulu`) ta kompara-s.

:::callout{type=tip}
Lembra `===` (identidadi) di Lisan 4? Gosi bu ta odja pamodi el importa: `karu1 === karu2` é `false` — es é **duas instánsias diferenti** na memória, mesmu ku mesmu dadus. Pa un klasse normal sima `Veikulu` (ka `data class`), `==` ta kai diretu na `===` — Kotlin ka jera igualdadi estrutural pa el.
:::

<SectionHeading variant="concept" seq={5}>Eransa: `open` i `override`</SectionHeading>

Gosi nu meste un tipu spesífiku di veikulu: un `Karu`, ku tudu propriedadis di `Veikulu` más un propriedadi novu, `numeruPortas`. En bez di kopia kódiku di `Veikulu` diretu, nu ta uza **eransa** (inheritance): `Karu` ta **erda** di `Veikulu`, i ta sobreskrebe (override) kumportamentu ki é diferenti.
