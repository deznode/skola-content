# Klasses, Propriedadis i Eransa

Imajina bu ta konstrui un sistema pa un negósiu di **Aluguer di Veikulu** na Santiago. Kada karu ki bu ten pa aluga tene se própriu matrikula i se própriu presu pa dia — i bu meste guarda es informason di manera organizadu, pa dipos kria reservas pa klientis.

Imajina un formadur di biskoitu (cookie-cutter): el ka é biskoitu, el é sô un moldi ki ta deskrebe kuzê forma i tamanhu kada biskoitu debe ten. Kada bez ki bu uza-l riba di massa, bu ta kria un biskoitu real, ku se própriu valor. Un `class` (klasse) na Kotlin ta funsiona parecidu: el é un moldi ki ta deskrebe kumo un `Veikulu` debe ser strutura — kuzê propriedadi el ten — ma el ka é un veikulu real. Kada bez ki bu uza mesmu klasse pa kria un objetu, sima `Veikulu("ST-45-AB", 4500.0)`, bu ta kria un **instánsia real**, ku se própriu matrikula i se própriu presu — mesmu kumo bu pode korta txeu biskoitu di mesmu formadur, kada un ku se própriu valor.

Na es lisan, nu ta kumesa ku un klasse `Veikulu`, dipos nu ta odja kumo `this` ta distingui un konfliktu di nomis, kumo un `data class` `Reserva` ta trata igualdadi i kópia di manera diferenti di un klasse normal, i pa fin kumo un klasse `Karu` pode erda di `Veikulu` ku eransa.

<SectionHeading variant="concept" seq={2}>Konstrutor Sekundáriu i Bloku init</SectionHeading>

Ás vezis bu meste izekuta kódiku **durante** kriason di un objetu — sima verifika si un valor é válidu — ou da **más di un manera** di kria un objetu. Kotlin ta da dos ferramenta pa kel: **bloku `init`** i **konstrutor sekundáriu**.

<SectionHeading variant="concept" seq={3}>Receiver i `this`</SectionHeading>

Kada `Veikulu` kriadu a partir di klasse é kumo un biskoitu spesífiku, kortadu di mesmu formadur. Ma kuandu un biskoitu meste "fala" di si mesmu — pa le ou muda si própriu valor — el meste un manera di dizê: "ami, kel biskoitu li spesífiku, ka kualker otu." Na Kotlin, kel referénsia ta txomadu `this` (na Python, `self`).

Kuandu bu skrebe un métodu dentu di un klasse, es métodu ta ben ku un objetu implísitu — **receiver** — ki é instánsia ki ta txoma métodu. Bu pode referensia kel objetu explisitamenti ku `this`. Normalmenti bu ka meste skrebe `this.` pamodi Kotlin já ta asumi-l, ma ten un kazu unde el é **obrigatóriu**: kuandu un parámetru di métodu tene mesmu nomi ki un propriedadi di klasse.

```kotlin
class Veikulu(
    val matrikula: String,
    var presuPerDia: Double,
) {
    fun atualizaPrecu(presuPerDia: Double) {
        println("Presu antigu di $matrikula: ${this.presuPerDia} ECV")
        this.presuPerDia = presuPerDia
        println("Presu novu di $matrikula: ${this.presuPerDia} ECV")
    }
}

fun main() {
    val karu1 = Veikulu("ST-45-AB", 4500.0)
    karu1.atualizaPrecu(5000.0)
}
// Presu antigu di ST-45-AB: 4500.0 ECV
// Presu novu di ST-45-AB: 5000.0 ECV
```

Dentu di `atualizaPrecu`, parámetru `presuPerDia` ta "skonde" propriedadi ku mesmu nomi. Si bu skrebe só `presuPerDia`, Kotlin ta intende ku é parámetru — ka propriedadi di objetu. Pa referensia **propriedadi di objetu**, bu meste ser explisitu: `this.presuPerDia`.

:::callout{type=tip}
Bu ka meste skrebe `this.` sénpri — Kotlin ta asumi-l automatikamenti kuandu ka ten konfuzon di nomi. Uza-l só kuandu, sima li riba, un parámetru ta skonde un propriedadi.
:::

<SectionHeading variant="concept" seq={4}>Data classes: `Reserva`</SectionHeading>

Gosi nu meste modela un **reserva** (booking) — kual klienti ta aluga kual veikulu, i pa kantu dia. Si `Reserva` fôr un klasse normal, nu tinha ki skrebe manualmenti métodu pa: kompara dos objetu (`equals`), jera un "impressan digital" úniku pa kada objetu (`hashCode`), mostra objetu di forma lejível (`toString`), i kria un kópia ku pikinoti alterason (`copy`) — kuatu métodu boilerplate, sô pamodi `Reserva` ta guarda dadus. Kotlin tene un modifikador `data` ki ta jera tudu es métodu **automatikamenti**, a partir di propriedadis ki bu dékrara na konstrutor primariu — sen bu skrebe nenhun kódiku ripitidu. `data class` ka ta troka konportamentu di klasse; el sô ta poupa bo di skrebe kódiku ki kompilador pode jera sózin:

```kotlin
class Veikulu(
    val matrikula: String,
    var presuPerDia: Double,
)

data class Reserva(
    val klienti: String,
    val veikulu: Veikulu,
    val dias: Int,
)

fun main() {
    val karu1 = Veikulu("ST-45-AB", 4500.0)
    val karu2 = Veikulu("ST-45-AB", 4500.0)   // mesmu valoris, otu instánsia

    val reserva1 = Reserva("Nando", karu1, 3)
    val reserva2 = Reserva("Nando", karu1, 3)   // mesmu veikulu (mesmu instánsia)
    val reserva3 = Reserva("Nando", karu2, 3)   // veikulu diferenti (mesmu valoris)

    println(reserva1)              // Reserva(klienti=Nando, veikulu=Veikulu@..., dias=3)
    println(reserva1 == reserva2)  // true  — mesmu klienti, mesmu veikulu, mesmu dias
    println(reserva1 == reserva3)  // false — karu1 i karu2 é instánsias diferenti

    val (klienti, veikulu, dias) = reserva1
    println("$klienti ta aluga veikulu ${veikulu.matrikula} pa $dias dia")

    val reserva4 = reserva1.copy(dias = 5)
    println(reserva4)              // Reserva(klienti=Nando, veikulu=Veikulu@..., dias=5)
}
```

Nota dos kuza importanti li. `reserva1 == reserva2` é `true` pamodi `Reserva` é `data class` — igualdadi ta kompara **valor** di kada propriedadi, ka referensia. Ma `reserva1 == reserva3` é `false`! Mesmu ku `karu1` i `karu2` tene mesmu `matrikula` i mesmu `presuPerDia`, es é **duas instánsias diferenti** di `Veikulu` — i `Veikulu` ka é `data class`, nton el ka tene igualdadi estrutural própriu. Un `data class` só ta kompara si própriu propriedadis; el ka ta muda kumo otus klasses (sima `Veikulu`) ta kompara-s.

:::callout{type=tip}
Lembra `===` (identidadi) di Lisan 4? Gosi bu ta odja pamodi el importa: `karu1 === karu2` é `false` — es é **duas instánsias diferenti** na memória, mesmu ku mesmu dadus. Pa un klasse normal sima `Veikulu` (ka `data class`), `==` ta kai diretu na `===` — Kotlin ka jera igualdadi estrutural pa el.
:::

<SectionHeading variant="concept" seq={5}>Eransa: `open` i `override`</SectionHeading>

Imajina árvori genealójika di Telma: el tene ojus kastanhu di se mai, i altura di se pai — el ka meste "inventa" kes karakterístika di zero, es ta ben diretu di se família. Ma Telma inda tene se própriu trasus úniku, sima se manera di optimiza kódiku.

**Eransa** (inheritance) na programasun ta funsiona parecidu: un **klasse fidju** (child class) ta erda tudu propriedadis i métodus di un **klasse pai** (parent class), sen meste kopia nenhun kódiku, i inda pode adisiona se própriu koza ou muda kel ki el erda. Gosi nu meste un tipu spesífiku di veikulu: un `Karu`, ku tudu propriedadis di `Veikulu` más un propriedadi novu di se própriu, `numeruPortas`. En bez di kopia kódiku di `Veikulu` diretu, `Karu` ta **erda** di `Veikulu` — `Veikulu` é klasse pai, `Karu` é klasse fidju — i ta sobreskrebe (override) kumportamentu ki é diferenti.

Ma ten un regra importanti na Kotlin: klasses i métodus é **`final`** pa default — ka pode ser erdadu ou sobreskrebe sen permison explisitu. Bu meste marka klasse i métodu ku `open`:

Ku `open` na se lugar, `Karu` pode erda di `Veikulu`, txoma se konstrutor — `: Veikulu(matrikula, presuPerDia)` — i sobreskrebe `mostraDetalis()` ku `override`:

```kotlin
open class Veikulu(
    val matrikula: String,
    var presuPerDia: Double,
) {
    open fun mostraDetalis() {
        println("Veikulu $matrikula — $presuPerDia ECV/dia")
    }
}

class Karu(
    matrikula: String,
    presuPerDia: Double,
    val numeruPortas: Int = 4,
) : Veikulu(matrikula, presuPerDia) {
    override fun mostraDetalis() {
        println("Karu $matrikula — $presuPerDia ECV/dia, $numeruPortas portas")
    }
}

fun main() {
    val veikulu: Veikulu = Karu("ST-45-AB", 4500.0)
    veikulu.mostraDetalis()
}
// Karu ST-45-AB — 4500.0 ECV/dia, 4 portas
```

:::callout{type=warning}
Skise di `open` ka ta kria un bug ki bu ta diskubri só dipos — el ta para kompilason imediatamenti, ku un erru klaru. Isu é diferenti di otus linguajens unde klasses pode ser erdadu pa default.
:::
