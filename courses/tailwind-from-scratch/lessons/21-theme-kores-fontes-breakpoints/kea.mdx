# @theme â€” Kores, Fontis, Breakpoints

Te li, nu uza `@theme` `n` vezes â€” pa `--breakpoint-3xl` (Lisan 14), `--spacing` (Lisan 6), `--animate-fade-up` (Lisan 20), `--container-*` (Lisan 15). Es lisan ta djunta tudu i mostra **manera kompletu di konfigura Tailwind v4**.

Mensajen prinsipal: `tailwind.config.js` Ă© **antigu**. v4 ta konfigura **direta na CSS**. Es ka Ă© sĂł mudansa di formatu â€” Ă© un sistema mĂˇs poderozu.

<SectionHeading variant="concept" seq={1}>Mudansa di Konseitu: v3 â†’ v4</SectionHeading>

### v3 â€” Konfigurasan kumo JavaScript

```js

module.exports = {
  theme: {
    extend: {
      colors: {
        brand: { 500: '#0284c7' }
      },
      fontFamily: {
        display: ['Crimson Pro', 'serif']
      }
    }
  }
}
```

### v4 â€” Konfigurasan kumo CSS

```html

  @theme {
    --color-brand-500: oklch(0.65 0.18 240);
    --font-display: "Crimson Pro", serif;
  }
</style>
```

**Kuandu bo defini `--color-brand-500`, klasis `bg-brand-500`, `text-brand-500`, `border-brand-500` etc. son jeradu otomatikamenti.**

Es Ă© **kontrĂˇriu di fluxu**: en bez di konfigura JavaScript ki jera CSS, bo skrebe CSS direta i Tailwind ta usa-l.

<SectionHeading variant="concept" seq={2}>Namespaces: Konvensan di Nomi</SectionHeading>

Cada tipu di token ten un namespace (prefixu). Kel ta determina ki utilidadis son jeradu:

| Namespace | Jera klasis |
|---|---|
| `--color-*` | `bg-*`, `text-*`, `border-*`, `ring-*`, `shadow-*` (kuandu uza `shadow-{kor}`), `accent-*`, etc. |
| `--font-*` | `font-{name}` |
| `--text-*` | `text-{name}` (tamanhu di fonti) |
| `--spacing` | `p-*`, `m-*`, `gap-*`, `w-*`, `h-*` (multiplu) |
| `--breakpoint-*` | Variantis responsivu (`sm:`, `md:`, etc.) |
| `--container-*` | Tamanhu di `container` i kontainer queries |
| `--radius-*` | `rounded-*` |
| `--shadow-*` | `shadow-*` |
| `--blur-*` | `blur-*` |
| `--animate-*` | `animate-*` |
| `--font-weight-*` | `font-*` (weights) |
| `--tracking-*` | `tracking-*` |
| `--leading-*` | `leading-*` |
| `--ease-*` | `ease-*` |

<SectionHeading variant="concept" seq={3}>`--color-*`: Kores ku OKLCH</SectionHeading>

v4 ta uza **OKLCH** kumo sistema interno (lembra Lisan 5). OKLCH Ă© mais perceptual ki RGB â€” kores ku mesmu "L" parese ten mesmu nivel di klaru.

```css
@theme {
  --color-brand-50: oklch(0.97 0.02 240);   /* muito klaru */
  --color-brand-100: oklch(0.93 0.04 240);
  --color-brand-200: oklch(0.85 0.08 240);
  --color-brand-300: oklch(0.75 0.12 240);
  --color-brand-400: oklch(0.65 0.15 240);
  --color-brand-500: oklch(0.55 0.18 240);  /* default â€” saturasaun max */
  --color-brand-600: oklch(0.45 0.18 240);
  --color-brand-700: oklch(0.38 0.16 240);
  --color-brand-800: oklch(0.30 0.14 240);
  --color-brand-900: oklch(0.22 0.10 240);
  --color-brand-950: oklch(0.15 0.06 240);  /* muito sukuru */
}
```

Bo pode tambĂ©n uza hex o rgb si OKLCH Ă© difisil:

```css
@theme {
  --color-brand-500: #0284c7;
  --color-brand-700: #0369a1;
}
```

(Ma OKLCH ten **vantajen real:** muda sĂł "L" ta jera tonalidadi konsistenti.)

Klasis jeradu otomatikamenti:

```html

  Tudu funsiona â€” Tailwind dezenha kel klasi base ofu tokens
</div>
```

<SectionHeading variant="concept" seq={4}>`--font-*`, `--breakpoint-*` i o restu</SectionHeading>

O mesmu padran ta aplika pa tudu os otu namespaces. Os fontis:

```html
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Crimson+Pro:wght@400;700&family=Inter:wght@400;500;700&display=swap">

<style type="text/tailwindcss">
  @theme {
    --font-sans: "Inter", system-ui, sans-serif;
    --font-display: "Crimson Pro", serif;
    --font-mono: "JetBrains Mono", monospace;
  }
</style>

<h1 class="font-display">Crimson Pro pa tĂ­tulu</h1>
<body class="font-sans">Inter pa textu kotidianu</body>
<code class="font-mono">JetBrains Mono pa kĂłdiku</code>
```

Repara: `--font-sans` override default. **Mas tudu utilidadi base ki uza `font-sans` (kumo `<body>` default) ta gosi uza Inter.**

Os breakpoints (lembra Lisan 14):

```css
@theme {
  --breakpoint-3xl: 1920px;
  --breakpoint-tv: 2560px;
}
```

Klasis `3xl:`, `tv:` funsiona otomatikamenti.

`--spacing` Ă© un token Ăşniku â€” muda-l i tudu skala di spacing ta muda djuntu (lembra Lisan 6):

```css
@theme {
  --spacing: 0.3rem;  /* tudu skala +20% mĂˇs larga */
}
```

`p-2`, `m-4`, `gap-6` â€” tudu son multiplu.

I os namespaces di stilu â€” `--radius-*`, `--shadow-*`, `--blur-*`, `--animate-*`:

```css
@theme {
  --radius-mega: 3rem;
  --shadow-card: 0 4px 20px -2px rgba(0, 0, 0, 0.1);
  --blur-extreme: 80px;
  --animate-fade-up: fade-up 0.6s ease-out forwards;  /* lembra Lisan 20 */
}
```

Jera klasis `rounded-mega`, `shadow-card`, `blur-extreme`, `animate-fade-up`.

<SectionHeading variant="concept" seq={5}>Override sen Extend</SectionHeading>

Na v3, bo tinha **dos opsan** na `tailwind.config.js`:

1. **`theme.colors.brand = ...`** â€” substitui tudu kores (perigozu)
2. **`theme.extend.colors.brand = ...`** â€” adisiona ku defaults (kazu kumun)

**Na v4, Ă© simples:** defini un token ka removi defaults. Defaults son sempri disponivel a menos ki bo override-l di forma eksplisitu.

```css
/* v4: --color-blue-500 default sempri ta funsiona */
/* sĂł si bo redefini, ai overridi */
@theme {
  --color-blue-500: red;  /* gosi bg-blue-500 ta da burmedju â€” perigozu */
}
```

Pa adisiona (kazu kumun), sĂł **defini nomi novu** â€” kel ka ta ten konflitu ku defaults:

```css
@theme {
  --color-brand-500: oklch(0.55 0.18 240);  /* novu */
}
```

`bg-blue-500` kontinua ta funsiona, `bg-brand-500` funsiona tambĂ©n.
