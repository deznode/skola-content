# Instala Python i Konfigura VS Code

Antis di kumesa programá, nu meste prepara nos ferramentas. É manera un kuzinheru ki ta organiza ingredientis antis di kumesa faze katxupa — bu meste tudu na lugar sertu! Na es lisan, nu ta instala Python, konfigura VS Code, kria un virtual environment, i kore nos primeru programa.

## Pasu 1: Instala Python

![Pasus di instalasan di Python na Windows, macOS i Linux](../../infographics/01-instala-python-plataformas.png)

Python é grátis i disponível pa tudu sistema operasional. Bai na **[python.org/downloads](https://www.python.org/downloads/)** i faze download di versan más resenti (Python 3.13+).

### Windows

1. Bai na [python.org/downloads](https://www.python.org/downloads/) i klika na **"Download Python 3.13.x"**
2. Abri fixeru ki bu faze download (`python-3.13.x-amd64.exe`)
3. **MUTU IMPORTANTI:** Marka txekbox **"Add python.exe to PATH"** na parti di baxu!
4. Klika **"Install Now"**
5. Spera instalasan termina i klika **"Close"**

:::callout{type=tip}
**O ki é PATH?** PATH é un lista di pastas undi sistema ta buska programas. Sen Python na PATH, te ki bu skrebe `python` na terminal, sistema ka ta atxa-l.
:::

:::callout{type=warning}
Si bu skise marka "Add python.exe to PATH", Python ka ta funsiona na terminal. Si bu instala sen marka, kore installer otu bez i skodje "Modify" → marka "Add to PATH".
:::

Pa verifika ki tudu ta funsiona, abri **Command Prompt** (o PowerShell) i skrebe:

```bash
python --version
```

Bu debe odja algu manera: `Python 3.13.2`

### macOS

1. Bai na [python.org/downloads](https://www.python.org/downloads/) i faze download di fixeru `.pkg` pa macOS
2. Abri fixeru `.pkg` i sigi instrusan di instalador
3. Klika **"Install"** i po bu password si pidi

Pa verifika, abri **Terminal** i skrebe:

```bash
python3 --version
```

:::callout{type=tip}
**Dika:** Na macOS, uza `python3` (ku 3) na lugar di `python`. macOS ta ben ku un Python antigu pré-instaladu, nton `python3` garanti ki bu ta uza versan novu.
:::

### Linux (Ubuntu/Debian)

Na maioria di distribuisons Linux, Python dja ta ben instaladu. Pa atualiza o instala:

```bash
sudo apt update
sudo apt install python3 python3-venv python3-pip
```

Verifika ku:

```bash
python3 --version
```

:::callout{type=tip}
**Dika:** Na Linux, é importanti instala tanbe `python3-venv` pamodi el ta permiti bu kria virtual environments. Sen kel pakoti, `python -m venv` ta da eru.
:::

## Pasu 2: Instala VS Code

**VS Code** (Visual Studio Code) é un editor di kódiku grátis di Microsoft. É editor más uzadu na mundu, ku suporti exelenti pa Python.

1. Bai na **[code.visualstudio.com](https://code.visualstudio.com/)** i faze download pa bu sistema
2. Instala sigindi instrusan normal (Next → Next → Install)
3. Abri VS Code

### Instala Extensan di Python

Extensan di Python ta da-bu:
- Sintaxi highlighting (korís na kódiku)
- Autocomplete intelizenti
- Detesan di eru na tempu real
- Debugger integradu

Pa instala:

1. Abri VS Code
2. Klika na íkoni di **Extensions** na barra lateral (o primi `Ctrl+Shift+X`)
3. Skrebe **"Python"** na barra di buska
4. Skodje extensan **"Python"** di Microsoft (el ta mostru *ms-python.python*)
5. Klika **"Install"**

Extensan ta instala automátikamenti **Pylance** (pa autocomplete) i **Python Debugger**.

## Pasu 3: Kria Bu Primeru Projetu

Gosi nu ta kria un pasta pa nos primeru projetu Python. Es é un bom ábitu — kada projetu deve tene si própriu pasta.

### Kria Pasta di Projetu

```bash
# Windows (na Command Prompt)
mkdir meu-primeru-python
cd meu-primeru-python

# macOS/Linux (na Terminal)
mkdir meu-primeru-python
cd meu-primeru-python
```

### Abri na VS Code

```bash
code .
```

Es komandu ta abri VS Code ku pasta atual kumo projetu. Si `code` ka ta funsiona, abri VS Code manualmenti i uza **File → Open Folder** pa skodje bu pasta.

## Pasu 4: Kria Virtual Environment (.venv)

Un **virtual environment** é un espasu izoladu pa kada projetu Python. Pamodi é importanti? Imagina:

- **Projetu A** presiza di biblioteka versan 1.0
- **Projetu B** presiza di mesmu biblioteka ma versan 2.0
- Sen virtual environment, un ta kebra otu!

Ku virtual environment, kada projetu tene si própriu "mundu" separadu.

### Kria .venv

Na terminal di VS Code (abri ku `` Ctrl+` ``), skrebe:

```bash
# Windows
python -m venv .venv

# macOS/Linux
python3 -m venv .venv
```

Es ta kria un pasta `.venv` dentu di bu projetu ku un kópia isoladu di Python.

### Ativa .venv

Antis di uza virtual environment, bu presiza **ativa** el:

```bash
# Windows (Command Prompt)
.venv\Scripts\activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1

# macOS/Linux
source .venv/bin/activate
```

Kuandu .venv ta ativu, bu ta odja `(.venv)` na kumessu di prompt:

```
(.venv) C:\Users\Nilton\meu-primeru-python>
```

Kel `(.venv)` ta indika ki tudu pakoti ki bu instala gosi ta bai pa es projetu so, ka pa sistema interu.

:::callout{type=warning}
**(Windows PowerShell):** Si bu odja eru di "execution policy", roda es komandu primeru:
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```
Dipos tenta ativa .venv otu bez.
:::

### VS Code i .venv

VS Code é intelizenti — kuandu el deteta un pasta `.venv`, el ta propoe automátikamenti pa uza kel Python. Odja na barra di status (parti di baxu) — el deve mostru algu manera:

```
Python 3.13.2 ('.venv')
```

Si ka ta mostra, klika na versan di Python na barra di status i skodje interpreter (kel programa Python ki ta roda bu kódiku) ki ta stá dentu di `.venv`.

## Pasu 5: Nha Primeru Programa!

Gosi si! Nu ta skrebe nos primeru programa Python.

### Kria Fixeru

1. Na VS Code, klika na **New File** (o `Ctrl+N`)
2. Salva kumo **`ola.py`** (File → Save, o `Ctrl+S`)

:::callout{type=tip}
**Dika:** Fixeru Python senpri tene extensan **`.py`**. Kel é manera Python sabe ki é un programa pa roda.
:::

### Skrebe Kódiku

Skrebe es kódiku na fixeru `ola.py`:

:::callout{type=tip}
**Dika:** Es `f` antis di aspras (`f"..."`) é un **f-string** — el ta permiti po variáveis dentu di tekstu ku `{nomi}`. Bu ta prende más na lisan 7 (Strings).
:::

```python
# Nha primeru programa Python!
# Kriadu pa aprendedores di Skola.dev

nomi = input("Kumo é bu nomi? ")
ilha = input("Bu é di kal ilha? ")

print(f"Ola, {nomi}! Ben-vindu a Python!")
print(f"{ilha} é un ilha sabi! Morabeza! 🇨🇻")
print("Nos ta kriá djunta!")
```

### Roda Programa

Tene 3 maneras di roda bu programa:

**Manera 1: Botan Play** (más fásil)
- Klika na botan ▶️ (play) na kantu superiór direitu di VS Code

**Manera 2: Terminal**
```bash
# Sertifika ki .venv ta ativu!
python ola.py
```

**Manera 3: Ataju di Tekladu**
- Primi `Ctrl+F5` (roda sen debugger)

Bu debe odja algu manera:

```
Kumo é bu nomi? Cesária
Bu é di kal ilha? São Vicente
Ola, Cesária! Ben-vindu a Python!
São Vicente é un ilha sabi! Morabeza! 🇨🇻
Nos ta kriá djunta!
```

**Parabéns! Bu dja skrebe i roda bu primeru programa Python!** 🎉

## Pasu 6: Komandus Esensiais pa Lembra

Aki tene un rezimu di komandus ki bu ta uza senpri:

### Virtual Environment

```bash
# Kria virtual environment
python -m venv .venv          # Windows
python3 -m venv .venv         # macOS/Linux

# Ativa
.venv\Scripts\activate         # Windows (cmd)
.venv\Scripts\Activate.ps1    # Windows (PowerShell)
source .venv/bin/activate      # macOS/Linux

# Dezativa (kuandu bu termina)
deactivate
```

### pip (Instalador di Pakotis)

Un **pakoti** é un biblioteka di kódiku ki otu pesoa skrebe pa bu uza. `pip` é programa ki ta instala-l dentu di bu `.venv`.

```bash
# Instala un pakoti dentu di bu .venv
pip install requests
```

:::callout{type=tip}
**Pa más sobri pip:** Lisan 21 (Ambiente Virtual i pip) ta mostra `pip list`, `pip freeze` i `requirements.txt` ku detalhe.
:::

### Roda Programa

```bash
# Roda un fixeru Python
python ola.py               # Windows
python3 ola.py              # macOS/Linux (fora di .venv)
```

:::callout{type=tip}
**Dika:** Kuandu `.venv` ta ativu, bu pode uza `python` (sen 3) na tudu sistema — virtual environment dja sabe kal versan pa uza.
:::

## Erus Kumuns i Soluson

| Eru | Kauza | Soluson |
|------|-------|---------|
| `python ka ta axa` (Windows) | PATH ka ta konfiguradu | Roda installer otu bez, marka "Add to PATH" |
| `python: command not found` (macOS) | Uza `python3` na lugar | Skrebe `python3` o ativa .venv primeru |
| PowerShell execution policy | Restrisan di seguransa | `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` |
| `pip install` ta da eru | .venv ka ta ativu | Verifika ki `(.venv)` ta na prompt |
| VS Code ka ta rekunhesi Python | Interpreter ka seleksionadu | Klika na barra di status → skodje .venv interpreter |
