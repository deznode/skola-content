# LĂ© Errus (Tracebacks)

Tudu programador ta atxa errus â€” disdi ki ta kumesa atĂ© ki ta vira espesialista. Diferensa Ă© ki un programador sperienti **sabi lĂ©** mensajen di erru i rezolve rĂˇpidu. Na es lisun, nĂ´s ta prende a lĂ© tracebacks di Python manera un detetivi â€” di baxu pa sima â€” pa diskubri exatamenti o ki korreu mal i undi.

Errus ka Ă© sinĂˇl di frakasu. Ă‰ sinĂˇl di ki bo ta tenta. NĂ´s ta kriĂˇ djunta, i parti di kria Ă© erĂˇ!

## O ki Ă© un Traceback?

Kuandu Python atxa un problema na bo kĂłdiku, e ta para i mostra un **traceback** â€” un mensajen ki ta splika o ki korreu mal i undi. Odja es ezemplu:

```python
# fixeru: kalkula.py
def kalkula_media(notas):
    total = sum(notas)
    media = total / len(notas)
    return media

notas_di_ana = []
rezultadu = kalkula_media(notas_di_ana)
print(f"MĂ©dia: {rezultadu}")
```

Kuandu bo roda es kĂłdiku, Python ta mostra:

```
Traceback (most recent call last):
  File "kalkula.py", line 7, in <module>
    rezultadu = kalkula_media(notas_di_ana)
                ~~~~~~~~~~~~~~~~~~~~~~~~~~~
  File "kalkula.py", line 3, in kalkula_media
    media = total / len(notas)
            ~~~~~~^~~~~~~~~~~~
ZeroDivisionError: division by zero
```

Es parsesi komplikadu, ma ka Ă©! NĂ´s ta prende lĂ© el pasu a pasu.

## Regra di Oru: LĂ© di Baxu pa Sima

Segridu mĂˇs importanti: **sempri kumesa lĂ© di Ăşltima linha**. Ă‰ la ki resposta ta stĂˇ.

### Pasu 1: Ăšltima Linha â€” O ki korreu mal

```
ZeroDivisionError: division by zero
```

Es linha tene **dĂ´s parti**:
- **`ZeroDivisionError`** â€” tipu di erru (nomi di exception)
- **`division by zero`** â€” mensajen ki ta splika problema

Tradusun: "Bo ta tenta dividi pa zero â€” Python ka pode fazi kel."

### Pasu 2: Linhas di Sima â€” Undi korreu mal

```
  File "kalkula.py", line 3, in kalkula_media
    media = total / len(notas)
            ~~~~~~^~~~~~~~~~~~
```

Es ta dize-bu:
- **Fixeru:** `kalkula.py`
- **Linha:** 3
- **Funsun:** `kalkula_media`
- **KĂłdiku:** `media = total / len(notas)`
- **Seta `^`:** aponta direitamenti pa `/` â€” Ă© operasun ki faya!

### Pasu 3: Subi mĂˇs â€” Kemi txama kemi

```
  File "kalkula.py", line 7, in <module>
    rezultadu = kalkula_media(notas_di_ana)
```

Es ta mostra ki linha 7 Ă© undi bo txama funsan ki faya. `<module>` signifika ki ta na nĂ­vel prinsipĂˇl di programa (ka dentru di un funsan).

:::callout{type=tip}
**Dika:** Pensa na traceback manera un pilha di pratus â€” Ăşltima pratu (Ăşltima linha) Ă© undi pratu kibra. Ma bo precisa odja pratus di baxu pa kompriendi pamodi tudu kai.
:::

## Seta `^` â€” Bo Melhor Amigu (Python 3.11+)

Disdi Python 3.11, errus tene un **seta `^`** ki ta aponta exatamenti pa parti di kĂłdiku ki ta kauza problema. Es Ă© un melhoria enormi!

```python
# Ezemplu: erru na espresun kompleksu
presu = {"katchupa": 350, "grogue": 200}
total = presu["katchupa"] + presu["ponche"]
```

```
Traceback (most recent call last):
  File "loja.py", line 3, in <module>
    total = presu["katchupa"] + presu["ponche"]
                                ~~~~~^^^^^^^^^^
KeyError: 'ponche'
```

Odja undi `^` ta aponta: `presu["ponche"]` â€” Ă© exatamenti kel parti ki ta kauza erru! "ponche" ka ezisti na disionĂˇriu `presu`.

MĂˇs un ezemplu:

```python
nomis = ["Maria", "JoĂŁo", "Ana"]
print(nomis[5])
```

```
Traceback (most recent call last):
  File "nomis.py", line 2, in <module>
    print(nomis[5])
          ~~~~~^^^
IndexError: list index out of range
```

Seta `^` ta mostra ki `nomis[5]` Ă© problema â€” lista so tene 3 elementus (Ă­ndisi 0, 1, 2), ma bo ta pidi Ă­ndisi 5.

## 5 Exceptions MĂˇs Kumun

Es Ă© errus ki bo ta atxa mĂˇs frekventi kuandu bo ta kumesa. Konhesi kes i bo ta rezolve problemus mutu mĂˇs rĂˇpidu.

### 1. `SyntaxError` â€” Erru di Sintaxi

Python ka kompriende o ki bo skrebi. Ă‰ manera un frazi ku gramĂˇtika eradu.

```python
# Skesi dĂ´s pontus (:) despuĂ©s di if
idadi = 20
if idadi > 18
    print("Adultu")
```

```
  File "testa.py", line 3
    if idadi > 18
                 ^
SyntaxError: expected ':'
```

**Kauzas kumun:**
- Skesi `:` dipos di `if`, `for`, `while`, `def`, `class`
- Skesi fexĂˇ parĂ©ntesis `)`, kĂłlxeti `]`, o xavi `}`
- Uza `=` (atribuisun) na lugar di `==` (komparasun) na kondisionais

:::callout{type=tip}
**Dika:** `SyntaxError` Ă© Ăşniku erru ki Python ta atxa **antis** di roda programa. Tudu otru errus ta aparesi kuandu programa ta roda.
:::

### 2. `NameError` â€” Nomi ka Ezisti

Bo ta uza un variĂˇvel ki ka foi kriadu inda.

```python
print(mensajen)
```

```
Traceback (most recent call last):
  File "testa.py", line 1, in <module>
    print(mensajen)
          ^^^^^^^^
NameError: name 'mensajen' is not defined
```

**Kauzas kumun:**
- VariĂˇvel ka foi atribuĂ­du (skesi `mensajen = "Ola"`)
- Erru di ortografia: `mesnajen` na lugar di `mensajen`
- Uza variĂˇvel fora di si scope (ex.: dentru di un funsan vs fora)

```python
# Korresun: defini variĂˇvel antis di uza
mensajen = "Ola, mundu!"
print(mensajen)
```

### 3. `TypeError` â€” Tipu Eradu

Bo ta mistura tipus ki ka Ă© kompatĂ­vel.

```python
idadi = 25
mensajen = "Bo tene " + idadi + " anu"
```

```
Traceback (most recent call last):
  File "testa.py", line 2, in <module>
    mensajen = "Bo tene " + idadi + " anu"
               ~~~~~~~~~~~~^~~~~~
TypeError: can only concatenate str (not "int") to str
```

Seta `^` ta mostra exatamenti undi string ta bate ku int!

**Kauzas kumun:**
- Djunta string ku int: `"Ola" + 5` (uza `"Ola" + str(5)` o f-string)
- Txama algu ki ka Ă© funsan: `x = 5; x()` (TypeError: 'int' is not callable)
- Manda argumentu eradu pa funsan

```python
# Korresun: uza f-string
idadi = 25
mensajen = f"Bo tene {idadi} anu"
print(mensajen)  # Bo tene 25 anu
```

### 4. `IndexError` â€” ĂŤndisi Fora di Rangu

Bo ta tenta asesa un posisun ki ka ezisti na lista.

```python
ilhas = ["Santiago", "SĂŁo Vicente", "Sal"]
print(ilhas[3])
```

```
Traceback (most recent call last):
  File "testa.py", line 2, in <module>
    print(ilhas[3])
          ~~~~~^^^
IndexError: list index out of range
```

Lista tene 3 elementus (Ă­ndisi 0, 1, 2), ma bo ta pidi Ă­ndisi 3.

**Kauzas kumun:**
- Skesi ki Ă­ndisi ta kumesa na **0** (ka na 1)
- Uza `len(lista)` kumo Ă­ndisi (Ăşltimo Ă­ndisi vĂˇlidu Ă© `len(lista) - 1`)
- Lista vĂˇziu i bo ta tenta asesa kualker Ă­ndisi

```python
# Korresun: verifika tamaĂ±u antis
ilhas = ["Santiago", "SĂŁo Vicente", "Sal"]
indisi = 3
if indisi < len(ilhas):
    print(ilhas[indisi])
else:
    print(f"ĂŤndisi {indisi} ka ezisti! Lista so tene {len(ilhas)} elementus.")
```

### 5. `KeyError` â€” Xavi ka Ezisti na DisionĂˇriu

Bo ta buska un xavi ki ka ezisti na disionĂˇriu.

```python
estudanti = {"nomi": "Kelvin", "idadi": 22}
print(estudanti["nota"])
```

```
Traceback (most recent call last):
  File "testa.py", line 2, in <module>
    print(estudanti["nota"])
          ~~~~~~~~~^^^^^^^^
KeyError: 'nota'
```

**Korresun: uza `.get()` pa evita erru**

```python
estudanti = {"nomi": "Kelvin", "idadi": 22}

# Manera seguru â€” retorna None si ka ezisti
nota = estudanti.get("nota")
print(nota)  # None

# Manera seguru ku valĂłr padraun
nota = estudanti.get("nota", "Sem nota")
print(nota)  # Sem nota
```

## Rezimu Vizual: 5 Exceptions

| Exception | Signifikadu | Ezemplu RĂˇpidu | Korresun Kumun |
|-----------|-------------|-----------------|----------------|
| `SyntaxError` | GramĂˇtika eradu | `if x > 5` (falta `:`) | Djunta `:`, `)`, `]` ki falta |
| `NameError` | VariĂˇvel ka defini | `print(x)` sem `x = ...` | Defini variĂˇvel o verifica ortografia |
| `TypeError` | Tipus inkompatĂ­vel | `"Ola" + 5` | Uza `str()`, `int()` o f-string |
| `IndexError` | ĂŤndisi fora di rangu | `lista[10]` ku 3 elementus | Verifika `len()` antis |
| `KeyError` | Xavi ka ezisti | `d["xavi"]` ki ka ezisti | Uza `.get(xavi, padraun)` |

## EstratĂ©jia di 3 Pasus pa Rezolvi Erru

Kuandu bo odja un erru, sigui sempri es 3 pasus:

```
1. LĂ‰  â†’  Ăšltima linha di traceback (tipu + mensajen)
2. ATXA â†’  Fixeru i nĂşmero di linha (seta ^ mostra lokalizasun)
3. PENSA â†’  "Pamodi Python ta pensa ki es Ă© un problema?"
```

Vamus prĂˇtika ku un ezemplu:

```python
# Programa ku erru â€” bo pode atxa?
lista_di_presus = [350, 200, 150, 400]
total = 0

for i in range(1, 5):
    total = total + lista_di_presus[i]

media = total / 4
print(f"MĂ©dia di presu: {media} ECV")
```

```
Traceback (most recent call last):
  File "presus.py", line 5, in <module>
    total = total + lista_di_presus[i]
                    ~~~~~~~~~~~~~~~^^^
IndexError: list index out of range
```

Aplika 3 pasus:
1. **LĂ‰:** `IndexError: list index out of range` â€” Ă­ndisi fora di rangu
2. **ATXA:** Linha 5, `lista_di_presus[i]` â€” seta mostra ki problema Ă© na asesu a lista
3. **PENSA:** `range(1, 5)` ta da `1, 2, 3, 4`. Ma `lista_di_presus[4]` ka ezisti pamodi Ă­ndisis ta bai di 0 a 3. Soluson: `range(len(lista_di_presus))` o `range(0, 4)`.

```python
# Korrejidu!
lista_di_presus = [350, 200, 150, 400]
total = 0

for i in range(len(lista_di_presus)):
    total = total + lista_di_presus[i]

media = total / len(lista_di_presus)
print(f"MĂ©dia di presu: {media} ECV")  # MĂ©dia di presu: 275.0 ECV
```

:::callout{type=tip}
**Dika:** Na futuru, bo ta prende ki `sum(lista_di_presus)` Ă© manera mĂˇs simples di soma un lista. Ma pa gosi, importanti Ă© kompriendi kumo lĂ© errus!
:::
