# Ambiente Virtual i pip

Imajina bu tene dos kuzinha: un pa prepara katxupa i otu pa prepara pastéis. Kada kuzinha tene ses própriu ingredientis i utensilhos — es ka ta mistura. **Ambientis virtual** na Python funsiona exatamenti asim: kada projetu tene ses própriu "kuzinha" ku ses pakotis, sen interferi ku otus projetus.

## Pamodi Ambientis Virtual É Esensial?

Sen ambiente virtual, tudu pakotu ki bo instala ta bai pa un lugar global — partilhadu pa TUDU bo projetus. Kel ta kria problemas sério:

```text
❌ SEN ambiente virtual:
┌──────────────────────────────────────┐
│ Python Global                        │
│                                      │
│ Projetu A presiza: requests 2.28     │
│ Projetu B presiza: requests 2.31     │
│                                      │
│ 💥 KONFLIKTU! Só un versan podi      │
│    sta instaladu di kada bez!         │
└──────────────────────────────────────┘

✅ KU ambiente virtual:
┌─────────────────┐  ┌─────────────────┐
│ Projetu A       │  │ Projetu B       │
│ .venv/          │  │ .venv/          │
│ requests 2.28   │  │ requests 2.31   │
│ flask 2.0       │  │ django 5.0      │
│                 │  │                 │
│ ✓ Izoladu!     │  │ ✓ Izoladu!     │
└─────────────────┘  └─────────────────┘
```

**Problemas reaís ki ta kontesi sen venv:**

- Bo atualiza un pakotu pa Projetu B i Projetu A ta kéba
- Bu ka sabe kual pakotis bo projetu realmenti presiza (tene 200 instaladu global!)
- Kuandu bo manda projetu pa kolega, el ka konsigi roda pamodi falta pakotis

## Fluxu Kompletu: Kria → Ativa → Instala → Freeze

:::callout{type=info}
Bo dja kria un `.venv` na **Lisan 02** — entun Pasu 1–2 li é un **rekapitulasan rápidu**. O ki é nóvu nes lisan ben más baxu: `freeze`, partilha ku otu pesoa, i `.gitignore`.
:::

### Pasu 1: Kria Ambiente Virtual

```bash
# Navega pa bo pasta di projetu
cd meu_projetu

# Kria ambiente virtual (konvensan: .venv)
python -m venv .venv
```

Kel komandu ta kria un pasta `.venv/` ku un kópia izoladu di Python i pip.

### Pasu 2: Ativa Ambiente Virtual

```bash
# macOS / Linux
source .venv/bin/activate

# Windows (cmd)
.venv\Scripts\activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

Kuandu ativadu, bo terminal ta mostra `(.venv)` na inísiu:

```bash
(.venv) $ python --version
Python 3.13.2
(.venv) $ pip list
Package    Version
---------- -------
pip        24.0
```

:::callout{type=tip}
Si bu ta uza VS Code, el ta deteta `.venv` automatikamenti i ta pidi pa selesioná kumo interpretador Python. Klika "Yes"!
:::

### Pasu 3: Instala Pakotis

```bash
# Gosi tudu ki bo instala ta bai SÓMENTI pa kel .venv
(.venv) $ pip install requests
(.venv) $ pip install flask
(.venv) $ pip install python-dotenv

# Verifica ki ta instaladu
(.venv) $ pip list
Package        Version
-------------- -------
certifi        2024.2.2
charset-normalizer 3.3.2
flask          3.0.2
requests       2.31.0
...
```

### Pasu 4: Jera requirements.txt

```bash
# "Freeze" tudu pakotu i ses versan pa un ficheru
(.venv) $ pip freeze > requirements.txt
```

Kel ficheru `requirements.txt` ta parsé ku kel li:

```text
certifi==2024.2.2
charset-normalizer==3.3.2
flask==3.0.2
requests==2.31.0
python-dotenv==1.0.1
```

### Pasu 5: Dezativa

```bash
# Kuandu bo akaba di trabadja
(.venv) $ deactivate
$ # Prompt volta pa normal -- bo sta fora di venv
```

## Partilha Projetu ku Otu Pesoa

Kuandu Ana kere roda bo projetu, el só presiza:

```bash
# 1. Klona ou kopia projetu
git clone https://github.com/nilton/gestor-notas.git
cd gestor-notas

# 2. Kria ses própriu ambiente virtual
python -m venv .venv

# 3. Ativa
source .venv/bin/activate  # (ó komandu Windows)

# 4. Instala tudu dependénsia a partir di requirements.txt
pip install -r requirements.txt

# 5. Pronto! Tudu pakotu instaladu ku versan serti
python main.py
```

:::callout{type=tip}
`pip install -r requirements.txt` ta ler kel ficheru i instala tudu pakotu listadu, ku versan exatu. É kumo un "reseta" pa bo projetu — garantia ki tudu pesoa ta uza mesmu ingredientis.
:::

## .gitignore: Nunka Manda .venv pa Git!

Pasta `.venv/` tene miláris di ficheru (Python interu + pakotis). El é **spesífiku** pa kada komputador i **ka debe bai pa git**:

```bash
# .gitignore
.venv/
__pycache__/
*.pyc
.env
```

```bash
# Struktura ki ta bai pa git:
gestor_notas/
├── .gitignore           ✓ ta bai
├── requirements.txt     ✓ ta bai (reseta di pakotis)
├── main.py              ✓ ta bai
├── modulus/             ✓ ta bai
│   ├── __init__.py
│   └── estudanti.py
├── .venv/               ✗ KA ta bai (na .gitignore)
└── .env                 ✗ KA ta bai (segredus!)
```

## Erus Komun i Soluson

### Eru 1: "python: command not found"

```bash
# Tenta python3 en bez di python (macOS/Linux)
python3 -m venv .venv
source .venv/bin/activate
```

### Eru 2: "No module named venv"

```bash
# Linux: presiza instala venv separadu
sudo apt install python3-venv
python3 -m venv .venv
```

### Eru 3: Skedji di ativa .venv dipos di kria

```bash
# Senpri ativa antis di instala!
# Si bo instala sen ativa, pakotu ta bai pa global ❌
source .venv/bin/activate   # PRIMERU
pip install requests        # DIPOS
```

### Eru 4: PowerShell blokeia aktivasan (Windows)

```powershell
# Si da eru di "execution policy":
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
# Dipos tenta di nóvu:
.venv\Scripts\Activate.ps1
```

### Eru 5: requirements.txt sta fora di sinku

```bash
# Si bo instala pakotu nóvu, atualiza requirements.txt!
pip install beautifulsoup4
pip freeze > requirements.txt   # Re-jera ficheru
```

## Fluxu Rezmidu (Txeat Sheet)

```bash
# === PROJETU NÓVU ===
mkdir meu_projetu && cd meu_projetu
python -m venv .venv
source .venv/bin/activate      # (ó .venv\Scripts\activate na Windows)
pip install requests flask
pip freeze > requirements.txt
echo ".venv/" >> .gitignore

# === PROJETU ESISTENTI (klonadu di git) ===
cd projetu_esistenti
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# === KUANDU AKABA ===
deactivate
```
