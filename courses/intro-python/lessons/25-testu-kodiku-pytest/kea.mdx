# Testu di Kódiku ku pytest

Imajina ki Carla ta kria un programa pa kalkula presu di katchupa na si restoranti na Mindelo. Programa ta funxiona perfeitu — até ki ela muda un funsan i, sem sabi, kibra kálkulu di diskonto. Klientis ta paga más di ki devia durante un simana interu antis di algen nota!

**Testus** ta evita es tipo di problema. É kódiku ki ta verifika ki bo otru kódiku ta funxiona koretu — automátikamenti, tudu vez ki bo kre.

## Pamodi Testa?

- **Kaptura bugs** antis di uzuáriu atxa
- **Konfiansa pa muda kódiku** — si testus ta pasa, nada kibra
- **Dokumentasun viva** — testus ta mostra kumo funsan deve konportá
- **Ta poupa tempu** — melhor testu automátiku na 1 segundu ki testu manuál na 5 minutu

## assert — Funsamentu di Tudu Testu

`assert` ta verifika ki un kondisun é verdadeiru. Si ka é, Python ta lansa `AssertionError`:

```python
# assert básiku
assert 2 + 2 == 4          # Pasa — nada akonteси
assert "ola".upper() == "OLA"  # Pasa

# Kuandu assert faya:
assert 2 + 2 == 5          # AssertionError!
assert 2 + 2 == 5, "Matimátika kibra!"  # AssertionError: Matimátika kibra!
```

`assert` é simples: si kondisun é `True`, tudu ben. Si é `False`, testu fayo.

## Instala pytest

pytest é ferramenta di testu más populár di Python. Instala na bo ambiente virtual:

```bash
# Ativa bo ambiente virtual primeru!
pip install pytest

# Verifika
pytest --version
```

## Primeiru Testu

Vamus kria un funsan i testa el. Primeru, nôs kria un fixeru ku funsan:

**`kalkuladora.py`:**

```python
def soma(a: float, b: float) -> float:
    """Soma dôs numerus"""
    return a + b

def subtrai(a: float, b: float) -> float:
    """Subtrai b di a"""
    return a - b

def multiplika(a: float, b: float) -> float:
    """Multiplika dôs numerus"""
    return a * b

def dividi(a: float, b: float) -> float | None:
    """Dividi a por b. Retorna None si b é zero."""
    if b == 0:
        return None
    return a / b
```

Gosi, kria un fixeru di testu. **Regra importanti:** nomi ten ki kumesa ku `test_` o akaba ku `_test.py`:

**`test_kalkuladora.py`:**

```python
from kalkuladora import soma, subtrai, multiplika, dividi

def test_soma_numerus_pozitivu():
    assert soma(2, 3) == 5

def test_soma_numerus_negativu():
    assert soma(-1, -1) == -2

def test_soma_ku_zero():
    assert soma(7, 0) == 7

def test_subtrai():
    assert subtrai(10, 3) == 7
    assert subtrai(5, 5) == 0
    assert subtrai(0, 5) == -5

def test_multiplika():
    assert multiplika(3, 4) == 12
    assert multiplika(0, 100) == 0
    assert multiplika(-2, 3) == -6

def test_dividi_normál():
    assert dividi(10, 2) == 5.0
    assert dividi(7, 2) == 3.5

def test_dividi_por_zero():
    assert dividi(10, 0) is None
    assert dividi(0, 0) is None
```

## Roda Testus

## Kuandu Testu Fayo

Vamus introduzi un bug pa odja o ki akonteси:

```python
# Bug proposital na kalkuladora.py
def soma(a: float, b: float) -> float:
    return a * b  # BUG! Ta multiplika na lugar di soma!
```

```bash
pytest -v

# test_kalkuladora.py::test_soma_numerus_pozitivu FAILED
# test_kalkuladora.py::test_soma_ku_zero FAILED
#
# FAILURES
# _________________ test_soma_numerus_pozitivu _________________
#
#     def test_soma_numerus_pozitivu():
# >       assert soma(2, 3) == 5
# E       assert 6 == 5
# E        +  where 6 = soma(2, 3)
#
# ==================== 2 failed, 5 passed in 0.02s ====================
```

pytest ta mostra:
- **Undi** fayo (fixeru + funsan di testu)
- **O ki** esperava (`== 5`)
- **O ki** resebeu (`6`)
- **Kemu** ta txama (`soma(2, 3)`)

Es informasun ta leva bo direitamenti pa bug!

## Testa Funsoens di Lisun Anteriu

Vamus testa funsoens ki nôs dja kria na kursу:

**`utils.py`:**

```python
def fatoriál(n: int) -> int:
    """Kalkula fatoriál di n (n!)"""
    if n < 0:
        raise ValueError("Ka pode kalkula fatoriál di numeru negativu!")
    if n == 0 or n == 1:
        return 1
    return n * fatoriál(n - 1)

def é_palindromu(tekstu: str) -> bool:
    """Verifika si tekstu é un palíndromu"""
    limpu = tekstu.lower().replace(" ", "")
    return limpu == limpu[::-1]

def konverti_temperatura(valór: float, di: str, pa: str) -> float | None:
    """Konverti entre Celsius, Fahrenheit, i Kelvin"""
    di = di.upper()
    pa = pa.upper()

    if di == pa:
        return valór

    # Primeru konverti pa Celsius
    if di == "F":
        celsius = (valór - 32) * 5 / 9
    elif di == "K":
        celsius = valór - 273.15
    elif di == "C":
        celsius = valór
    else:
        return None

    # Depois konverti di Celsius pa destinu
    if pa == "C":
        return celsius
    elif pa == "F":
        return celsius * 9 / 5 + 32
    elif pa == "K":
        return celsius + 273.15
    return None
```

**`test_utils.py`:**

```python
import pytest
from utils import fatoriál, é_palindromu, konverti_temperatura

# === Testus di fatoriál ===

def test_fatoriál_zero():
    assert fatoriál(0) == 1

def test_fatoriál_un():
    assert fatoriál(1) == 1

def test_fatoriál_sinku():
    assert fatoriál(5) == 120  # 5! = 5*4*3*2*1 = 120

def test_fatoriál_dez():
    assert fatoriál(10) == 3_628_800

def test_fatoriál_negativu_da_erru():
    """Verifika ki numeru negativu ta lansa ValueError"""
    with pytest.raises(ValueError):
        fatoriál(-1)

# === Testus di é_palindromu ===

def test_palindromu_simples():
    assert é_palindromu("ana") is True
    assert é_palindromu("ovo") is True

def test_palindromu_ku_maiúskula():
    assert é_palindromu("Ana") is True
    assert é_palindromu("Ovo") is True

def test_palindromu_ku_espasu():
    assert é_palindromu("a man a") is True

def test_ka_palindromu():
    assert é_palindromu("maria") is False
    assert é_palindromu("python") is False

# === Testus di konverti_temperatura ===

def test_celsius_pa_fahrenheit():
    assert konverti_temperatura(0, "C", "F") == 32.0
    assert konverti_temperatura(100, "C", "F") == 212.0

def test_fahrenheit_pa_celsius():
    assert konverti_temperatura(32, "F", "C") == 0.0
    assert konverti_temperatura(212, "F", "C") == 100.0

def test_celsius_pa_kelvin():
    assert konverti_temperatura(0, "C", "K") == 273.15
    assert konverti_temperatura(-273.15, "C", "K") == 0.0

def test_mesmu_unidadi():
    assert konverti_temperatura(50, "C", "C") == 50

def test_unidadi_inválidu():
    assert konverti_temperatura(50, "X", "C") is None
```

```bash
pytest test_utils.py -v

# test_utils.py::test_fatoriál_zero PASSED
# test_utils.py::test_fatoriál_un PASSED
# test_utils.py::test_fatoriál_sinku PASSED
# test_utils.py::test_fatoriál_dez PASSED
# test_utils.py::test_fatoriál_negativu_da_erru PASSED
# test_utils.py::test_palindromu_simples PASSED
# ... tudu PASSED!
```

:::callout{type=tip}
**Dika:** `pytest.raises(ValueError)` é un forma espesiál di testa ki un funsan **deve** lansa un erru. Si funsan ka lansa ValueError, testu fayo!
:::

## Pensamentu Test-Driven (TDD Leve)

TDD signifika skrebi testus **antis** di skrebi kódiku. Siklu é:

```
1. SKREBI TESTU  →  (fayo pamodi funsan ka ezisti inda)
2. IMPLEMENTA    →  (skrebi kódiku mínima pa testu pasa)
3. VERIFIKA      →  (roda pytest — tudu verdе?)
4. REPETI        →  (adisiona más testus, melhora kódiku)
```

### Ezemplu: Validador di Email (TDD)

**Pasu 1 — Skrebi testus primeru:**

```python
# test_validador.py
from validador import valida_email

def test_email_válidu():
    assert valida_email("maria@gmail.com") is True
    assert valida_email("joao.silva@uni-cv.edu") is True

def test_email_sem_aroba():
    assert valida_email("mariagmail.com") is False

def test_email_sem_domíniu():
    assert valida_email("maria@") is False

def test_email_váziu():
    assert valida_email("") is False

def test_email_ku_espasu():
    assert valida_email("maria @gmail.com") is False
```

```bash
pytest test_validador.py
# ERROR: ModuleNotFoundError — 'validador' ka ezisti inda!
```

**Pasu 2 — Implementa pa testus pasa:**

```python
# validador.py
def valida_email(email: str) -> bool:
    """Validasun básiku di email"""
    if not email or " " in email:
        return False
    if "@" not in email:
        return False
    partis = email.split("@")
    if len(partis) != 2:
        return False
    uzuáriu, domíniu = partis
    if not uzuáriu or not domíniu:
        return False
    if "." not in domíniu:
        return False
    return True
```

**Pasu 3 — Verifika:**

```bash
pytest test_validador.py -v
# tudu PASSED! ✓
```

## Konvensoens di Nomi pa Testu

| Regra | Ezemplu |
|-------|---------|
| Fixeru kumesa ku `test_` | `test_kalkuladora.py` |
| O fixeru akaba ku `_test.py` | `kalkuladora_test.py` |
| Funsun di testu kumesa ku `test_` | `def test_soma():` |
| Nomi deskritivi | `test_soma_ku_numerus_negativu` |

:::callout{type=tip}
**Dika:** Nomi di testu deve dizi **o ki ta testa** i **rezultadu speradu**. `test_dividi_por_zero_retorna_none` é mutu melhor ki `test_1`.
:::

## Struktura di Projetu ku Testus

```
nha_projetu/
├── kalkuladora.py
├── utils.py
├── validador.py
├── test_kalkuladora.py
├── test_utils.py
└── test_validador.py
```

Pa projetиs más grandi, testus pode fikа na un pasta separadu:

```
nha_projetu/
├── src/
│   ├── kalkuladora.py
│   ├── utils.py
│   └── validador.py
└── tests/
    ├── test_kalkuladora.py
    ├── test_utils.py
    └── test_validador.py
```

## Rezumu

- **assert** — verifika ki un kondisun é verdadeiru; si ka é, testu fayo
- **pytest** — ferramenta ki ta diskubri i roda tudu funsoens `test_*` automátikamenti
- **Nomi di ficheru:** `test_*.py` o `*_test.py`; funsoens: `def test_*()`
- **Roda:** `pytest` (tudu) o `pytest -v` (verbozu ku nomi di kada testu)
- **`pytest.raises(Error)`** — testa ki un funsan lansa un exsesun espesífiku
- **Pensamentu TDD:** skrebi testu primeru → implementa → verifika → repeti
- **Testus é investimentu** — más tempu gosi, menus bugs dipos!
